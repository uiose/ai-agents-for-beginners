# AI 에이전트 초보자용 - 학습 가이드 및 강좌 요약

이 가이드는 "AI Agents for Beginners" 강좌의 요약을 제공하며 AI 에이전트를 구축하기 위한 주요 개념, 프레임워크 및 설계 패턴을 설명합니다.

## 1. AI 에이전트 소개

**AI 에이전트란 무엇인가?**
AI 에이전트는 도구, 지식 및 메모리에 대한 접근을 제공함으로써 대형 언어 모델(LLMs)의 기능을 확장하는 시스템입니다. 학습 데이터에 기반하여 텍스트만 생성하는 표준 LLM 챗봇과 달리, AI 에이전트는:
- **환경을 인지**합니다(센서 또는 입력을 통해).
- **문제를 해결하기 위해 추론**합니다.
- **환경을 변경하기 위해 행동**합니다(액추에이터 또는 도구 실행을 통해).

**에이전트의 주요 구성 요소:**
- **환경**: 에이전트가 작동하는 공간(예: 예약 시스템).
- **센서**: 정보를 수집하는 메커니즘(예: API 읽기).
- **액추에이터**: 동작을 수행하는 메커니즘(예: 이메일 보내기).
- **브레인(LLM)**: 어떤 행동을 취할지 계획하고 결정하는 추론 엔진.

## 2. 에이전틱 프레임워크

강좌에서는 에이전트를 구축하기 위해 **Microsoft Agent Framework (MAF)** 와 **Azure AI Foundry Agent Service V2** 를 사용합니다:

| Component | Focus | Best For |
|-----------|-------|----------|
| **Microsoft Agent Framework** | 에이전트, 도구, 워크플로우를 위한 통합 Python/C# SDK | 도구 사용이 포함된 에이전트, 다중 에이전트 워크플로우 및 프로덕션 패턴 구축에 적합합니다. |
| **Azure AI Foundry Agent Service** | 관리형 클라우드 런타임 | 내장된 상태 관리, 관찰성 및 신뢰 기능을 갖춘 안전하고 확장 가능한 배포. |

## 3. 에이전틱 설계 패턴

설계 패턴은 에이전트가 신뢰성 있게 문제를 해결하도록 구조화하는 데 도움을 줍니다.

### **도구 사용 패턴** (수업 4)
이 패턴은 에이전트가 외부 세계와 상호작용할 수 있게 합니다.
- **개념**: 에이전트에는 사용 가능한 함수와 그 매개변수의 목록인 "스키마"가 제공됩니다. LLM은 사용자의 요청에 따라 어떤 도구를 호출할지와 어떤 인수로 호출할지를 결정합니다.
- **흐름**: 사용자 요청 -> LLM -> **도구 선택** -> **도구 실행** -> LLM(도구 출력 포함) -> 최종 응답.
- **사용 사례**: 실시간 데이터(날씨, 주가) 조회, 계산 수행, 코드 실행 등.

### **계획 패턴** (수업 7)
이 패턴은 에이전트가 복잡하고 여러 단계의 작업을 해결할 수 있게 합니다.
- **개념**: 에이전트는 상위 목표를 더 작은 하위 작업들의 연속으로 분해합니다.
- **접근법**:
  - **작업 분해**: "여행 계획"을 "항공편 예약", "호텔 예약", "렌터카"로 분할합니다.
  - **반복적 계획**: 이전 단계의 출력에 따라 계획을 재평가합니다(예: 항공편이 만석이면 다른 날짜를 선택).
- **구현**: 종종 구조화된 계획(예: JSON)을 생성하는 "플래너" 에이전트가 있으며, 이는 다른 에이전트가 실행합니다.

## 4. 설계 원칙

에이전트를 설계할 때는 세 가지 차원을 고려하세요:
- **공간**: 에이전트는 사람과 지식을 연결해야 하며, 접근 가능하면서도 눈에 거슬리지 않아야 합니다.
- **시간**: 에이전트는 과거로부터 학습하고, 현재에 관련된 알림을 제공하며, 미래에 적응해야 합니다.
- **핵심**: 불확실성을 수용하되 투명성과 사용자 제어를 통해 신뢰를 구축하세요.

## 5. 주요 강의 요약

- **Lesson 1**: 에이전트는 단순한 모델이 아니라 시스템입니다. 그들은 인지하고, 추론하며, 행동합니다.
- **Lesson 2**: Microsoft Agent Framework는 도구 호출과 상태 관리의 복잡성을 추상화합니다.
- **Lesson 3**: 투명성과 사용자 제어를 염두에 두고 설계하십시오.
- **Lesson 4**: 도구는 에이전트의 "손"입니다. LLM이 도구를 어떻게 사용하는지 이해하려면 스키마 정의가 중요합니다.
- **Lesson 7**: 계획은 에이전트의 "집행 기능"으로 복잡한 워크플로우를 처리할 수 있게 합니다.
