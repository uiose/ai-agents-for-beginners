# Агенты ИИ для начинающих - Учебное руководство и краткое содержание курса

This guide provides a summary of the "AI Agents for Beginners" course and explains key concepts, frameworks, and design patterns for building AI Agents.

## 1. Введение в агентов ИИ

**Что такое агенты ИИ?**
Агенты ИИ — это системы, которые расширяют возможности Large Language Models (LLMs), предоставляя им доступ к **инструментам**, **знаниям** и **памяти**. В отличие от стандартного чат-бота на базе LLM, который лишь генерирует текст на основе данных обучения, агент ИИ может:
- **Воспринимать** свою среду (через сенсоры или входные данные).
- **Рассуждать** о том, как решить задачу.
- **Действовать**, чтобы изменить среду (через исполнительные механизмы или выполнение инструментов).

**Ключевые компоненты агента:**
- **Окружающая среда**: Пространство, в котором работает агент (например, система бронирования).
- **Сенсоры**: Механизмы для сбора информации (например, чтение API).
- **Актуаторы**: Механизмы для выполнения действий (например, отправка электронной почты).
- **Мозг (LLM)**: Двигатель рассуждений, который планирует и решает, какие действия выполнять.

## 3. Паттерны проектирования агентов

Паттерны проектирования помогают структурировать работу агентов для надежного решения задач.

### **Tool Use Pattern** (Урок 4)
Этот паттерн позволяет агентам взаимодействовать с внешним миром.
- **Концепция**: агенту предоставляется «схема» (список доступных функций и их параметров). LLM решает, *какой* инструмент вызвать и с *какими* аргументами на основе запроса пользователя.
- **Поток**: User Request -> LLM -> **Выбор инструмента** -> **Выполнение инструмента** -> LLM (с выводом инструмента) -> Окончательный ответ.
- **Сценарии использования**: получение данных в реальном времени (погода, цены на акции), выполнение вычислений, запуск кода.

### **Planning Pattern** (Урок 7)
Этот паттерн позволяет агентам решать сложные многозадачные задачи.
- **Концепция**: агент разбивает задачу высокого уровня на последовательность более мелких подзадач.
- **Подходы**:
  - **Декомпозиция задач**: разбиение «Спланировать поездку» на «Забронировать рейс», «Забронировать отель», «Арендовать автомобиль».
  - **Итеративное планирование**: переоценка плана на основе результатов предыдущих шагов (например, если рейс заполнен, выбрать другую дату).
- **Реализация**: часто включает агента "Planner", который генерирует структурированный план (например, JSON), который затем выполняют другие агенты.

## 4. Принципы проектирования

При проектировании агентов учитывайте три измерения:
- **Пространство**: Агенты должны связывать людей и знания, быть доступными, но ненавязчивыми.
- **Время**: Агенты должны учиться на *прошлом*, предоставлять релевантные подсказки в *настоящее* и адаптироваться для *будущего*.
- **Ядро**: Принимайте неопределенность, но устанавливайте доверие через прозрачность и контроль со стороны пользователя.

## 5. Краткое содержание ключевых уроков

- **Lesson 1**: Агенты — это системы, а не просто модели. Они воспринимают, рассуждают и действуют.
- **Lesson 2**: Microsoft Agent Framework абстрагирует сложность вызова инструментов и управления состоянием.
- **Lesson 3**: Проектируйте с учетом прозрачности и контроля со стороны пользователя.
- **Lesson 4**: Инструменты — это «руки» агента. Определение схемы критично для того, чтобы LLM понимала, как их использовать.
- **Lesson 7**: Планирование — это «исполнительная функция» агента, позволяющая ему справляться со сложными рабочими процессами.
