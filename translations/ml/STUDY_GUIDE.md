# AI ഏജന്റുകൾ ആരംഭക്കാർക്കുള്ള - പഠന ഗൈഡ് & കോഴ്‌സ് സംഗ്രഹം

ഈ ഗൈഡ് "AI Agents for Beginners" കോഴ്സിന്റെ ഒരു സംഗ്രഹം നൽകുകയും AI ഏജന്റുകൾ നിർമ്മിക്കാൻ പ്രധാന ആശയങ്ങൾ, ഫ്രെയിംവർക്കുകൾ, ഡിസ്‌ൈൻ പാറ്റേണുകൾ എന്നിവ വിശദീകരിക്കുകയും ചെയ്യുന്നു.

## 1. AI ഏജന്റുകളിലേക്ക് പരിചയം

**AI ഏജന്റുകൾ എന്താണു?**
AI ഏജന്റുകൾ വലിയ ഭാഷാ മോഡലുകളുടെ (LLMs) കഴിവുകൾ വികസിപ്പിച്ച് അവയ്ക്ക് **ടൂളുകൾ**, **അറിയിപ്പ്**, மற்றும் **മെമ്മറി** എന്നിവയുടെ ആക്‌സസ് നൽകുന്ന സിസ്റ്റങ്ങളാണ്. മറ്റൊരു സാധാരണ LLM ചാറ്റ്ബോട്ട് ട്രെയിനിംഗ് ഡാറ്റയുടെ അടിസ്ഥാനത്തിൽ മാത്രമേ ടെക്സ്റ്റ് അനുസരിച്ച് സൃഷ്ടിക്കുകയുള്ളു; ഒരു AI ഏജന്റ് സാധിക്കാനാകുന്നത്:
- **പരീക്ഷിക്കുക** തന്റെ പരിസ്ഥിതി (സെൻസറുകൾ അല്ലെങ്കിൽ ഇൻപുട്ടുകളിലൂടെ).
- **കാരണം കണ്ടെത്തുക** ഒരു പ്രശ്നം എങ്ങനെ പരിഹരിക്കാമെന്ന്.
- **പ്രവർത്തിക്കുക** പരിസ്ഥിതിയെ മാറ്റാൻ (ആക്ടുവേറ്ററുകൾ അല്ലെങ്കിൽ ടൂൾ നിർവ്വഹണം വഴി).

**ഒരു ഏജന്റിന്റെ പ്രധാന ഘടകങ്ങൾ:**
- **Environment**: ഏജന്റ് പ്രവർത്തിക്കുന്ന സ്ഥലം (ഉദാഹരണത്തിന്, ഒരു ബുക്കിംഗ് സിസ്റ്റം).
- **Sensors**: വിവരം ശേഖരിക്കുന്ന യന്ത്രങ്ങൾ (ഉദാഹരണത്തിന്, ഒരു API വായിക്കുക).
- **Actuators**: പ്രവർത്തനങ്ങൾ നടത്താനുള്ള യന്ത്രങ്ങൾ (ഉദാഹരണത്തിന്, ഒരു ഇമെയിൽ അയക്കൽ).
- **Brain (LLM)**: പ്ലാൻ ചെയ്ത് ഏത് പ്രവർത്തനങ്ങൾ എടുക്കണമെന്നു തീരുമാനിക്കുന്ന കാരണശേഷി എൻജിൻ.

## 2. Agentic Frameworks

കോഴ്സ് **Microsoft Agent Framework (MAF)** നെ **Azure AI Foundry Agent Service V2** നെ ഉപയോഗിച്ചാണ് ഏജന്റുകൾ നിർമ്മിക്കുന്നത്:

| Component | Focus | Best For |
|-----------|-------|----------|
| **Microsoft Agent Framework** | ഐക്യപ്പെടുത്തിയ Python/C# SDK ഏജന്റുകൾ, ടൂളുകൾ, വർക്ക്‌ഫ്ലോകുകൾ എന്നിവയ്‌ക്കായി | ടൂളുകളോട് ചേർന്ന് പ്രവർത്തിക്കുന്ന ഏജന്റുകൾ, മൾട്ടി-ഏജന്റ് വർക്‌ഫ്ലോകുകൾ, പ്രൊഡക്ഷൻ പാറ്റേണുകൾ എന്നിവ നിർമ്മിക്കാൻ. |
| **Azure AI Foundry Agent Service** | മാനേജുചെയ്ത ക്ലൗഡ് റൺടൈം | നിർമിത സ്റ്റേറ്റ് മാനേജ്‌മെന്റ്, ഓബ്സർവബിലിറ്റി, ട്രസ്റ്റ് എന്നിവയോടുകൂടിയ സുരക്ഷിതവും സ്കേലബിൾമായ ഡിപ്ലോയ്‌മെന്റ്. |

## 3. Agentic Design Patterns

ഡിസ്‌ൈൻ പാറ്റേണുകൾ ഏജന്റുകൾ എങ്ങനെ വിശ്വാസപരമായി പ്രശ്നങ്ങൾ പരിഹരിക്കാവുന്ന വിധം ഘടനപ്പെടുത്താമെന്ന് സഹായിക്കുന്നു.

### **Tool Use Pattern** (Lesson 4)
ഈ പാറ്റേൺ ഏജന്റുകൾക്ക് പുറത്തുള്ള ലോകവുമായി ഇടപഴകാൻ സാധ്യമാക്കുന്നു.
- **Concept**: ഏജന്റിന് ലഭ്യമാക്കിയിരിക്കുന്ന ഫങ്ഷനുകളും അവയുടെ പാരാമീറ്ററുകളും ഉള്ള ഒരു "schema" (സൂചിപ്പിച്ച ലിസ്റ്റ്) ദായ ഉപയോഗിച്ച് നൽകപ്പെടുന്നു. ഉപയോക്താവിന്റെ അഭ്യർത്ഥനയുടെ അടിസ്ഥാനത്തിൽ LLM എത്രയും *എന്ത്* ടൂൾ വിളിക്കണം എന്നും *എന്ത്* arguments ഉപയോഗിക്കണം എന്നും തീരുമാനിക്കുന്നു.
- **Flow**: ഉപയോക്തൃ അഭ്യർത്ഥന -> LLM -> **ടൂൾ തെരഞ്ഞെടുപ്പ്** -> **ടൂൾ നിർവ്വഹണം** -> LLM (ടൂൾ ഔട്ട്പുട്ടോടുകൂടെ) -> അവസാന പ്രതികരണം.
- **Use Cases**: റിയൽ-ടൈം ഡാറ്റ (വാതാവരണം, സ്റ്റോക്ക് വിലകൾ) നേടൽ, കണക്കുകൾ നടത്തൽ, കോഡ് നിർവ്വഹിക്കൽ.

### **Planning Pattern** (Lesson 7)
ഈ പാറ്റേൺ ഏജന്റുകൾക്ക് ബഹുജടി, ബഹു-ഘട്ടമായ ടാസ്കുകൾ പരിഹരിക്കാൻ സഹായിക്കുന്നു.
- **Concept**: ഏജന്റ് ഒരു ഉയർന്ന തലത്തിലുള്ള ലക്ഷ്യത്തെ ছোট ചുവടുകൾ ആയി വിഭജിക്കുന്നു.
- **Approaches**:
  - **Task Decomposition**: "ഒരു യാത്ര പ്ലാൻ ചെയ്യുക" എന്നതിനെ "ഫ്ലൈറ്റ് ബുക്ക് ചെയ്യുക", "ഹോട്ടലിന് ബുക്ക് ചെയ്യുക", "കാർ വാടകയ്ക്ക് എടുക്കുക" എന്നിവയായി വിഭജിക്കൽ.
  - **Iterative Planning**: മുൻവകയിലുള്ള ഘട്ടങ്ങളുടെ ഔട്ട്പുട്ട് അടിസ്ഥാനമാക്കി പ്ലാൻ വീണ്ടും വിലയിരുത്തൽ (ഉദാഹരണത്തിന്, ഫ്ലൈറ്റ് പൂർണ്ണമാണെങ്കിൽ വ്യത്യസ്തമായ തീയതി തിരഞ്ഞെടുക്കുക).
- **Implementation**: പതിവായി ഒരു "Planner" ഏജന്റ് ഘടനയുള്ള പ്ലാൻ (ഉദാഹരണത്തിന് JSON) നിർമ്മിച്ച് അത് മറ്റ് ഏജന്റുകൾ നടപ്പാക്കുന്നു.

## 4. Design Principles

ഏജന്റുകൾ രൂപകൽപ്പന ചെയ്യുമ്പോൾ മൂന്ന് വിചാരണങ്ങൾ പരിഗണിക്കുക:
- **Space**: ഏജന്റുകൾ ആളുകളെയും അറിവിനെയും ബന്ധിപ്പിക്കണം, പ്രവേശനയോഗ്യവും പക്ഷേ അപ്രതിരോധ്യവുമാകണം.
- **Time**: ഏജന്റുകൾ *ഭൂതകാലത്തിൽ* നിന്ന് പഠിക്കണം, *ഇപ്പോൾ* പ്രസക്തമായ നഡ്‌ജുകൾ നൽകണം, και *ഭാവിക്ക്* അനുയോജ്യമായി രൂപാന്തരം വരുത്തണം.
- **Core**: അസ്പഷ്ടതကို സ്വീകരിക്കുക പക്ഷേത کاربرانക്ക് വ്യക്തതയും നിയന്ത്രണവും നൽകിയാണ് വിശ്വാസം സ്ഥാപിക്കുക.

## 5. Summary of Key Lessons

- **Lesson 1**: ഏജന്റുകൾ മോഡലുകളിൽ മാത്രം അല്ല, സമർത്ഥതയുള്ള സിസ്റ്റങ്ങളാണ്. അവ പരിശോധന, കാരണനിർണം ഒപ്പം പ്രവർത്തനം ചെയ്യുന്നു.
- **Lesson 2**: Microsoft Agent Framework ടൂൾ കോള് ചെയ്യലിന്റെയും സ്റ്റേറ്റ് മാനേജ്മെന്റിന്റെയും സങ്കീർണത abstraസ്ട് ചെയ്യുന്നു.
- **Lesson 3**: വ്യക്തതയും ഉപയോക്തൃ നിയന്ത്രണവും മനസ്സിൽവെച്ച് രൂപകൽപ്പന ചെയ്യുക.
- **Lesson 4**: ടൂളുകൾ ഏജന്റിന്റെ "ചെറുത്തുകളാണ്". സ്കീമ നിർവചനം LLMയ്ക്ക് അവയെ എങ്ങനെ ഉപയോഗിക്കാമെന്ന് മനസ്സിലാക്കാൻ നിർണായകമാണ്.
- **Lesson 7**: പ്ലാനിംഗ് ഏജന്റിന്റെ "എക്സിക്യൂട്ടീവ് ഫംഗ്ഷൻ" ആണ്, ഇത് അവന് ക്ലിഷ്ടമായ വർക്‌ഫ്ലോകൾ കൈകാര്യം ചെയ്യാൻ സഹായിക്കുന്നു.
