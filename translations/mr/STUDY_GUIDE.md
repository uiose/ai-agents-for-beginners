# AI एजंट्स सुरुवातीसाठी - अभ्यास मार्गदर्शक व कोर्स सारांश

हा मार्गदर्शक "AI Agents for Beginners" कोर्सचा सारांश देतो आणि AI एजंट्स तयार करण्यासाठी महत्त्वाच्या संकल्पना, फ्रेमवर्क आणि डिझाइन पॅटर्न समजावून सांगतो.

## 1. AI एजंट्स परिचय

**AI एजंट्स काय आहेत?**
AI एजंट्स ही अशी प्रणाली आहेत जी मोठ्या भाषा मॉडेल्स (LLMs) च्या क्षमतांमध्ये वाढ करतात, त्यांना **टूल्स**, **ज्ञान**, आणि **मेमरी** ऍक्सेस देऊन. फक्त प्रशिक्षण डेटावरून मजकूर निर्माण करणाऱ्या सामान्य LLM च्या उलट, एक AI एजंट:
- **परिसराचे निरीक्षण करतो** (सेन्सर किंवा इनपुटद्वारे).
- **समस्येवर विचार करतो** कसे सोडवायचे.
- **वातावरण बदलण्यासाठी क्रिया करतो** (ऍक्चुएटर्स किंवा टूल एक्झिक्युशनद्वारे).

**एजंटचे मुख्य घटक:**
- **Environment**: ज्या जागेत एजंट कार्य करतो ती जागा (उदा., बुकिंग सिस्टीम).
- **Sensors**: माहिती गोळा करण्याची पद्धत (उदा., API वाचन).
- **Actuators**: क्रिया करण्याची पद्धत (उदा., ईमेल पाठवणे).
- **Brain (LLM)**: विचार करण्याचे इंजिन जे योजना आखते आणि कोणत्या क्रिया घ्यायच्या हे ठरवते.

## 2. एजंटिक फ्रेमवर्क्स

हा कोर्स एजंट तयार करण्यासाठी **Microsoft Agent Framework (MAF)** आणि **Azure AI Foundry Agent Service V2** वापरतो:

| घटक | लक्ष | सर्वोत्तम उपयोग |
|-----------|-------|----------|
| **Microsoft Agent Framework** | एजंट्स, टूल्स, आणि वर्कफ्लोजसाठी एक संयुक्त Python/C# SDK | टूल्ससह एजंट तयार करणे, मल्टी-एजंट वर्कफ्लोज्, आणि प्रॉडक्शन पॅटर्नसाठी. |
| **Azure AI Foundry Agent Service** | व्यवस्थापित क्लाउड रनटाइम | अंतर्निर्मित स्टेट मॅनेजमेंट, ऑब्जर्व्हिबिलिटी, आणि ट्रस्टसह सुरक्षित, स्केलेबल डिप्लॉयमेंट. |

## 3. एजंटिक डिझाइन पॅटर्न

डिझाइन पॅटर्न एजंट कसे कार्य करतात हे संरचित करण्यास मदत करतात जेणेकरून समस्या विश्वासार्हपणे सोडवता येतील.

### **टूल वापरण्याचा पॅटर्न** (धडा 4)
हा पॅटर्न एजंट्सना बाह्य जगाशी संवाद साधण्यास सक्षम करतो.
- **संकल्पना**: एजंटला "schema" दिले जाते (उपलब्ध फंक्शन्स आणि त्यांच्या पॅरामीटर्सची यादी). वापरकर्त्याच्या विनंतीनुसार LLM ठरवते की *कोणते* टूल कॉल करायचे आणि *कुठल्या* अर्ग्युमेंट्ससह.
- **प्रवाह**: User Request -> LLM -> **Tool Selection** -> **Tool Execution** -> LLM (with tool output) -> Final Response.
- **वापर प्रकरणे**: रिअल-टाइम डेटा मिळवणे (हवामान, स्टॉक किंमती), गणना करणे, कोड अंमलात आणणे.

### **योजना पॅटर्न** (धडा 7)
हा पॅटर्न एजंट्सना जटिल, बहु-टप्प्याच्या कार्ये सोडवण्यास सक्षम करतो.
- **संकल्पना**: एजंट उच्च-स्तरीय उद्दिष्ट लहान उप-कार्यांच्या सिक्वेन्समध्ये विभागतो.
- **पध्दती**:
  - **कार्य विभाजन**: "यात्रेची योजना" ला "फ्लाइट बुक करा", "हॉटेल बुक करा", "कार भाड्याने घ्या" अशा उप-कार्यात विभागणे.
  - **पुनरावृत्ती नियोजन**: मागील टप्प्यांच्या आउटपुटनुसार योजना पुन्हा मुल्यमापन करणे (उदा., जर फ्लाइट भरला असेल तर वेगळी तारीख निवडा).
- **अंमलबजावणी**: अनेकदा यामध्ये एक "Planner" एजंट असतो जो संरचित योजना (उदा., JSON) तयार करतो आणि नंतर इतर एजंट्सद्वारे ती अंमलात आणली जाते.

## 4. डिझाइन तत्त्वे

एजंट डिझाइन करताना तीन पैलू विचारात घ्या:
- **अवकाश**: एजंट लोक आणि ज्ञान यांना जोडायला हवेत, प्रवेशयोग्य असावेत परंतु त्रासदायक नसावेत.
- **काल**: एजंटांनी *भूतकाळ*ातून शिकणे, *सद्य* काळात योग्य सूचनांसाठी डावपेच करणे, आणि *भविष्य*साठी अनुकूल होणे आवश्यक आहे.
- **मूल**: अनिश्चिततेला स्वीकारा परंतु पारदर्शकता आणि वापरकर्त्याच्या नियंत्रणाद्वारे विश्वास प्रस्थापित करा.

## 5. मुख्य धड्यांचा सारांश

- **धडा 1**: एजंट्स ही प्रणाली आहेत, फक्त मॉडेल नाहीत. ते निरीक्षण करतात, विचार करतात, आणि क्रिया करतात.
- **धडा 2**: Microsoft Agent Framework हे टूल कॉलिंग आणि स्टेट मॅनेजमेंटची गुंतागुंत abstract करते.
- **धडा 3**: पारदर्शकता आणि वापरकर्ता नियंत्रण लक्षात घेऊन डिझाइन करा.
- **धडा 4**: टूल्स हे एजंटची "हात" आहेत. स्कीमा परिभाषा ही LLM साठी कशी वापरायची हे समजून घेण्यासाठी अत्यावश्यक आहे.
- **धडा 7**: नियोजन हे एजंटचे "कार्यकारी कार्य" आहे, जे त्याला जटिल वर्कफ्लोज् हाताळण्यास सक्षम करते.
