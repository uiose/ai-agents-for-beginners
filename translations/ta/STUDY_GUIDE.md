# ஆரம்பக்காரர்களுக்கான AI முகவரிகள் - ஆய்வு கையேடு & பாடத்திட்ட சுருக்கம்

இந்த கையேடு "அரம்பக்காரர்களுக்கான AI முகவரிகள்" பாடத்திட்டத்தின் சுருக்கத்தைக் கொண்டுள்ளது மற்றும் AI முகவரிகளை உருவாக்குவதற்கான முக்கியக் கருத்துக்கள், கட்டமைப்புகள் மற்றும் வடிவமைப்பு முறைமைகள் விளக்குகிறது.

## 1. AI முகவரிகளுக்கு அறிமுகம்

**AI முகவரிகள் என்றால் என்ன?**  
AI முகவரிகள் என்பது பெரிய மொழி மாதிரிகளின் (LLMs) திறன்களை **கருவிகள்**, **அறிவு**, மற்றும் **நினைவு** ஆகியவற்றின் அணுகுமுறையுடன் விரிவாக்கும் முறைகள். சாதாரண LLM உரையாடல் இயந்திரம் பயிற்சி தரவின்படி உரையை உருவாக்குவதையே செய்கின்றது; ஆனால் AI முகவர் கூடுதலாக:  
- **உட்புற சூழலை உணர்ந்து** (உருவி அல்லது உள்ளீடுகள் மூலம்)  
- **சிக்கலை தீர்க்க வரைகோள் இடிந்து**  
- **சூழலை மாற்ற வேலை செய்கிறது** (கூறிகளின் செயல்படுதல் மூலம்)  

**முகவரியின் முக்கிய கூறுகள்:**  
- **சூழல்**: முகவர் செயற்படும் இடம் (எ.கா., முன்பதிவு அமைப்பு).  
- **உருவிகள்**: தகவல்களை சேகரிப்பதற்கான கருவிகள் (எ.கா., API வாசித்தல்).  
- **செயலிகள்**: நடவடிக்கைகள் மேற்கொள்ளும் கருவிகள் (எ.கா., மின்னஞ்சல் அனுப்பல்).  
- **மூளை (LLM)**: நடவடிக்கைகளை திட்டமிடும் மற்றும் தீர்மானிக்கும் காரணி இயங்கியல்.  

## 2. முகவர் கட்டமைப்புகள்

இந்த பாடத்தில், முகவரிகள் உருவாக்க **Microsoft Agent Framework (MAF)** மற்றும் **Azure AI Foundry Agent Service V2** பயன்படுத்தப்படுகிறது:

| கூறு | கவனம் | சிறந்த பயன்பாடு |
|-----------|-------|----------|
| **Microsoft Agent Framework** | முகவரிகளுக்கும் கருவிகளுக்கும், பணிக்கட்டமைப்புகளுக்கும் ஒருமைத்த Python/C# SDK | கருவிகளுடன் முகவரிகள் உருவாக்குதல், பல-முகவர் பணிக்கட்டமைப்புகள் மற்றும் தயாரிப்பு வடிவமைப்புகள். |
| **Azure AI Foundry Agent Service** | நிர்வகிக்கப்பட்ட மேக இயக்கத்தளமாகும் | பாதுகாப்பான, விரிவாக்கக்கூடிய இடைநிறைவு, மாநில மேலாண்மை, கண்காணிப்பு மற்றும் நம்பகத்தன்மையை வழங்குகிறது. |

## 3. முகவர் வடிவமைப்பு முறைமைகள்

வடிவமைப்புச் முறைகள் முகவரிகள் நம்பகமாக பிரச்சனைகளை தீர்க்க ஒருங்கிணைந்து செயல்பட வழிவகுக்கின்றன.

### **கருவி பயன்படுத்தும் முறை** (பாடம் 4)  
இந்த முறை முகவரிகளை வெளியுலகத்துடன் தொடர்பு கொள்ள உதவுகிறது.  
- **கருத்து**: முகவரிக்கு "வரைவு" (கிடைக்கும் செயல்பாடுகள் மற்றும் அவற்றின் அளவுருக்கள் பட்டியல்) வழங்கப்படுகிறது. LLM பயனர் கோரிக்கையின் அடிப்படையில் எது மற்றும் எவ்வாறு கருவியை பயன்படுத்துவது என்பதைக் கண்டெடுக்கிறது.  
- **பருவம்**: பயனர் கோரிக்கை -> LLM -> **கருவி தேர்வு** -> **கருவி செயல்பாடு** -> LLM (கருவி வெளியீட்டுடன்) -> இறுதி பதில்.  
- **பயன்பாடுகள்**: நேரடி தரவு பெறுதல் (வானிலை, பங்கு விலை), கணக்கீடுகள் செய்யல், குறியீடு இயக்குதல்.  

### **திட்டமிடும் முறை** (பாடம் 7)  
இந்த முறை, பல படிகளைக் கொண்ட பணிகளை முகவர்கள் தீர்க்க உதவுகிறது.  
- **கருத்து**: முகவர் ஒரு பெரிய இலக்கை சிறு துணைப் பணிகளாக உடைக்கின்றது.  
- **முறைமைகள்**:  
  - **பணி உடைத்தல்**: "ஒரு பயணம் திட்டமிடு" என்பதனை "விமானம் முன்பதிவு", "தங்கும் இடம் முன்பதிவு", "கார் வாடகை" என்ற துணை பணிகளாகப் பிரிதல்.  
  - **மீண்டும் ஆய்வு செய்வது**: முன் படிகளின் முடிவுகளின் அடிப்படையில் திட்டத்தை மீண்டும் மதிப்பீடு (எ.கா., விமானம் நிரம்பி இருந்தால் வேறு தேதியெடுத்து).  
- **ஏற்கப்பட்டல்**: பல நேரங்களில் "திட்டமிடுபவர்" முகவர் கட்டமைப்பை அமைத்துக் கொடுக்கிறது (எ.கா., JSON வடிவில்), பிற முகவர்கள் அதை இயக்குகின்றனர்.  

## 4. வடிவமைப்பு கொள்கைகள்

முகவர்கள் வடிவமைக்கும் போது மூன்று பரிமாணங்களை கருத்தில் கொள்ள வேண்டும்:  
- **இடம்**: முகவர்கள் மனிதர்கள் மற்றும் தகவலுக்கு இணைப்பாக இருக்க வேண்டும்; அணுகக்கூடியதாகவும், தடையின்றி இருக்க வேண்டும்.  
- **நேரம்**: முகவர்கள் *கடந்த காலம்* இல் இருந்து கற்றுக் கொண்டு, *இப்போதிருக்கும்* பொருத்தமான ஊக்கங்கள் வழங்கி, *எதிர்காலத்திற்கு* தக்க முறையில் மாறி நிற்கும்.  
- **மூலம்**: அறியாமையை ஏற்றுக்கொண்டு, வெளிப்படைத்தன்மையும் பயனர் கட்டுப்பாடும் மூலம் நம்பிக்கையை உருவாக்க வேண்டும்.  

## 5. முக்கிய பாடங்களின் சுருக்கம்

- **பாடம் 1**: முகவர்கள் யதார்த்த அமைப்புகள்; அவர்கள் உணர்கிறார்கள், முடிவு செய்கிறார்கள், செயல்படுகின்றனர்.  
- **பாடம் 2**: Microsoft Agent Framework கருவி அழிப்பு மற்றும் மாநில மேலாண்மையின் சிக்கலைத் திறம்பட கையாள்கிறது.  
- **பாடம் 3**: வெளிப்படைத்தன்மை மற்றும் பயனர் கட்டுப்பாடுகளை கருத்தில் கொண்டு வடிவமைக்க வேண்டும்.  
- **பாடம் 4**: கருவிகள் முகவரியின் "கைகள்" போன்றவை. LLM அவற்றைப் பயன்படுத்துவதற்கான வரைவைப் புரிந்து கொள்வது அவசியம்.  
- **பாடம் 7**: திட்டமிடல் என்பது முகவரியின் "நிர்வாக செயல்பாடு", இது சிக்கலான பணிகள் தொடரைக் கையாள உதவும்.
