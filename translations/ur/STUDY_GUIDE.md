# AI ایجنٹس برائے ابتدائی افراد - مطالعہ رہنما اور کورس کا خلاصہ

یہ رہنما "AI ایجنٹس برائے ابتدائی افراد" کورس کا خلاصہ فراہم کرتی ہے اور AI ایجنٹس بنانے کے لیے اہم تصورات، فریم ورکس، اور ڈیزائن پیٹرنز کی وضاحت کرتی ہے۔

## 1. AI ایجنٹس کا تعارف

**AI ایجنٹس کیا ہیں؟**
AI ایجنٹس وہ نظام ہیں جو Large Language Models (LLMs) کی صلاحیتوں کو **اوزار**، **علم**، اور **یادداشت** تک رسائی دے کر بڑھاتے ہیں۔ معمول کے ایک LLM چیٹ بوٹ کے برعکس جو صرف تربیتی ڈیٹا کی بنیاد پر متن پیدا کرتا ہے، ایک AI ایجنٹ کر سکتا ہے:
- **محسوس کرے** اپنا ماحول (سینسرز یا ان پٹس کے ذریعے).
- **استدلال کرے** کہ مسئلہ کیسے حل کیا جائے۔
- **عمل کرے** تاکہ ماحول کو تبدیل کرے (ایکچیویٹرز یا ٹول کے عمل درآمد کے ذریعے).

**ایجنٹ کے اہم اجزاء:**
- **ماحول**: وہ جگہ جہاں ایجنٹ کام کرتا ہے (مثلاً ایک بکنگ سسٹم).
- **سینسرز**: معلومات جمع کرنے کے طریقے (مثلاً، API پڑھنا).
- **ایکچیویٹرز**: عمل انجام دینے کے طریقے (مثلاً، ای میل بھیجنا).
- **دماغ (LLM)**: وہ استدلالی انجن جو منصوبہ بناتا ہے اور فیصلہ کرتا ہے کہ کون سے عمل کرنے ہیں۔

## 2. ایجنٹک فریم ورکس

کورس ایجنٹس بنانے کے لیے **Microsoft Agent Framework (MAF)** کو **Azure AI Foundry Agent Service V2** کے ساتھ استعمال کرتا ہے:

| Component | Focus | Best For |
|-----------|-------|----------|
| **Microsoft Agent Framework** | ایجنٹس، ٹولز، اور ورک فلو کے لیے متحد Python/C# SDK | ٹولز کے ساتھ ایجنٹس بنانا، کثیر ایجنٹ ورک فلو، اور پروڈکشن پیٹرنز۔ |
| **Azure AI Foundry Agent Service** | Managed cloud runtime | اندرونی ریاستی انتظام، مشاہداتی صلاحیت اور ٹرسٹ کے ساتھ محفوظ، اسکیل ایبل ڈپلائمنٹ۔ |

## 3. ایجنٹک ڈیزائن پیٹرنز

ڈیزائن پیٹرنز اس بات میں مدد دیتے ہیں کہ ایجنٹس کس طرح کام کرتے ہیں تاکہ مسائل کو بااعتماد طریقے سے حل کیا جا سکے۔

### **ٹول استعمال کرنے کا پیٹرن** (سبق 4)
یہ پیٹرن ایجنٹس کو بیرونی دنیا کے ساتھ تعامل کرنے کے قابل بناتا ہے۔
- **تصور**: ایجنٹ کو ایک "schema" فراہم کی جاتی ہے (دستیاب فنکشنز اور ان کے پیرامیٹرز کی فہرست). LLM فیصلہ کرتا ہے کہ صارف کی درخواست کی بنیاد پر *کون سا* ٹول کال کرنا ہے اور *کس* دلائل کے ساتھ۔
- **بہاؤ**: صارف کی درخواست -> LLM -> **ٹول منتخب کرنا** -> **ٹول اجرا** -> LLM (ٹول کے آؤٹ پٹ کے ساتھ) -> حتمی جواب۔
- **استعمال کے کیسز**: ریئل ٹائم ڈیٹا حاصل کرنا (موسم، اسٹاک قیمتیں)، حساب کتاب کرنا، کوڈ چلانا۔

### **منصوبہ بندی کا پیٹرن** (سبق 7)
یہ پیٹرن ایجنٹس کو پیچیدہ، کثیر مرحلہ وار کام حل کرنے کے قابل بناتا ہے۔
- **تصور**: ایجنٹ ایک اعلیٰ سطحی ہدف کو چھوٹے ذیلی کاموں کے سلسلے میں تقسیم کرتا ہے۔
- **طریقہ کار**:
  - **Task Decomposition**: "سفر کا منصوبہ بنائیں" کو "فلائٹ بک کریں"، "ہوٹل بک کریں"، "کار کرائے پر لیں" میں تقسیم کرنا۔
  - **Iterative Planning**: پچھلے مرحلوں کے نتائج کی بنیاد پر منصوبے کا از سر نو جائزہ لینا (مثلاً، اگر فلائٹ فل ہے تو مختلف تاریخ منتخب کریں)۔
- **نفاذ**: اکثر اس میں ایک "Planner" ایجنٹ شامل ہوتا ہے جو ایک ساختہ منصوبہ (مثلاً، JSON) بناتا ہے جسے پھر دیگر ایجنٹس اجرا کرتے ہیں۔

## 4. ڈیزائن اصول

ایجنٹس ڈیزائن کرتے وقت تین جہتوں پر غور کریں:
- **مقام**: ایجنٹس لوگوں اور علم کو جوڑیں، قابل رسائی ہوں مگر خلل ڈالنے والے نہ ہوں۔
- **وقت**: ایجنٹس کو *ماضی* سے سیکھنا چاہیے، *اب* میں متعلقہ اشارے دینا چاہیے، اور *مستقبل* کے لیے خود کو ڈھالنا چاہیے۔
- **بنیادی**: غیر یقینی کو قبول کریں مگر شفافیت اور صارف کے کنٹرول کے ذریعے اعتماد قائم کریں۔

## 5. اہم اسباق کا خلاصہ

- **سبق 1**: ایجنٹس نظام ہیں، صرف ماڈل نہیں۔ وہ محسوس کرتے ہیں، استدلال کرتے ہیں، اور عمل کرتے ہیں۔
- **سبق 2**: Microsoft Agent Framework ٹول کالنگ اور ریاستی انتظام کی پیچیدگی کو آسان بناتا ہے۔
- **سبق 3**: ڈیزائن کرتے وقت شفافیت اور صارف کے کنٹرول کو ذہن میں رکھیں۔
- **سبق 4**: ٹولز ایجنٹ کے "ہاتھ" ہیں۔ اسکیما کی تعریف LLM کے لیے یہ سمجھنے میں ضروری ہے کہ انہیں کیسے استعمال کیا جائے۔
- **سبق 7**: منصوبہ بندی ایجنٹ کا "انتظامی فعل" ہے، جو اسے پیچیدہ ورک فلوؤں سے نمٹنے کے قابل بناتی ہے۔
