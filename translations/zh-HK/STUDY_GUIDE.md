# AI Agents for Beginners - 學習指南與課程摘要

本指南總結了「AI Agents for Beginners」課程，並說明建立 AI 代理的關鍵概念、框架與設計模式。

## 1. AI Agents 簡介

**甚麼是 AI Agents？**
AI Agents 是透過提供 **工具**、**知識** 和 **記憶** 來擴展大型語言模型 (LLMs) 能力的系統。與只根據訓練資料產生文字的標準 LLM 聊天機械人不同，AI Agent 可以：
- **感知** 其環境（通過感測器或輸入）。
- **推理** 如何解決問題。
- **行動** 以改變環境（透過執行工具或致動器）。

**代理的關鍵組成部分：**
- **環境**：代理運作的空間（例如：預訂系統）。
- **感測器**：蒐集資訊的機制（例如：讀取 API）。
- **致動器**：執行動作的機制（例如：發送電郵）。
- **大腦 (LLM)**：負責規劃與決定採取何種行動的推理引擎。

## 2. Agentic Frameworks

本課程使用 **Microsoft Agent Framework (MAF)** 與 **Azure AI Foundry Agent Service V2** 來建立代理：

| Component | Focus | Best For |
|-----------|-------|----------|
| **Microsoft Agent Framework** | 提供統一的 Python/C# SDK，用於代理、工具與工作流程 | 建構具有工具、多代理工作流程與生產模式的代理。 |

## 3. Agentic Design Patterns

設計模式有助於組織代理如何運作，以可靠地解決問題。

### **工具使用模式** (Lesson 4)
這個模式讓代理能與外部世界互動。
- **概念**：代理會被提供一個 "schema"（可用函數及其參數的清單）。LLM 會根據使用者的請求決定 *呼叫哪一個* 工具以及 *使用甚麼* 參數。
- **流程**：使用者請求 -> LLM -> **工具選擇** -> **工具執行** -> LLM (含工具輸出) -> 最終回應。
- **使用情境**：擷取即時資料（天氣、股票價格）、執行計算、執行程式碼。

### **規劃模式** (Lesson 7)
這個模式讓代理能解決複雜的多步驟任務。
- **概念**：代理將高階目標拆解為一連串較小的子任務。
- **方法**：
  - **任務分解**：將 "Plan a trip" 拆成 "Book flight"、"Book hotel"、"Rent car"。
  - **迭代式規劃**：根據前一步驟的輸出重新評估計劃（例如：若航班客滿，改選其他日期）。
- **實作**：通常包含一個 "Planner" 代理來產生結構化計劃（例如 JSON），然後由其他代理執行。

## 4. 設計原則

在設計代理時，考慮三個面向：
- **空間**：代理應該連結人與知識，可被取得但不造成干擾。
- **時間**：代理應該從 *過去* 學習，在 *當下* 提供相關提示，並為 *未來* 作出調整。
- **核心**：接受不確定性，但透過透明度與使用者控制建立信任。

## 5. 重要課程重點摘要

- **Lesson 1**：代理是系統，而不僅僅是模型。它們會感知、推理並行動。
- **Lesson 2**：Microsoft Agent Framework 抽象化了工具呼叫與狀態管理的複雜性。
- **Lesson 3**：在設計時要以透明度與使用者控制為優先。
- **Lesson 4**：工具是代理的「手」。定義 schema 對 LLM 理解如何使用工具至關重要。
- **Lesson 7**：規劃是代理的「執行功能」，使其能處理複雜的工作流程。
