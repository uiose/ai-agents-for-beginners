# عامل‌های هوش مصنوعی برای مبتدی‌ها - راهنمای مطالعه و خلاصه دوره

این راهنما خلاصه‌ای از دوره «عامل‌های هوش مصنوعی برای مبتدی‌ها» را ارائه می‌دهد و مفاهیم کلیدی، چارچوب‌ها و الگوهای طراحی ساخت عامل‌های هوش مصنوعی را توضیح می‌دهد.

## 1. مقدمه‌ای بر عامل‌های هوش مصنوعی

**عامل‌های هوش مصنوعی چیستند؟**  
عامل‌های هوش مصنوعی سیستم‌هایی هستند که قابلیت‌های مدل‌های زبان بزرگ (LLM ها) را با دسترسی به **ابزارها**، **دانش** و **حافظه** گسترش می‌دهند. برخلاف یک چت‌بات LLM استاندارد که فقط بر اساس داده‌های آموزشی متن تولید می‌کند، یک عامل هوش مصنوعی می‌تواند:  
- **محیط** خود را درک کند (از طریق حسگرها یا ورودی‌ها).  
- **استدلال** کند که چگونه یک مسئله را حل کند.  
- **عمل** کند تا محیط را تغییر دهد (از طریق عملگرها یا اجرای ابزار).  

**اجزای کلیدی یک عامل:**  
- **محیط**: فضای عملیاتی عامل (مثلاً یک سامانه رزرو).  
- **حسگرها**: مکانیزم‌های گردآوری اطلاعات (مثلاً خواندن یک API).  
- **عملگرها**: مکانیزم‌های انجام اقدامات (مثلاً ارسال ایمیل).  
- **مغز (LLM)**: موتور استدلال که برنامه‌ریزی و تصمیم‌گیری برای انجام اقدامات را انجام می‌دهد.

## 2. چارچوب‌های عاملی

در این دوره از **Microsoft Agent Framework (MAF)** همراه با **Azure AI Foundry Agent Service V2** برای ساخت عامل‌ها استفاده می‌شود:

| جزء | تمرکز | بهترین کاربرد |
|-----------|-------|----------|
| **Microsoft Agent Framework** | SDK یکپارچه پایتون/C# برای عامل‌ها، ابزارها و جریان‌های کاری | ساخت عامل‌ها با ابزارها، جریان‌های کاری چندعاملی و الگوهای تولیدی. |
| **Azure AI Foundry Agent Service** | اجرای ابری مدیریت شده | استقرار امن و مقیاس‌پذیر با مدیریت وضعیت، قابلیت رصد و اطمینان داخلی. |

## 3. الگوهای طراحی عاملی

الگوهای طراحی به ساختاردهی نحوه عملکرد عامل‌ها برای حل مسئله به صورت قابل اطمینان کمک می‌کنند.

### **الگوی استفاده از ابزار** (درس ۴)  
این الگو به عامل‌ها اجازه می‌دهد با دنیای بیرون تعامل داشته باشند.  
- **مفهوم**: به عامل یک «طرح‌واره» (لیستی از توابع در دسترس و پارامترهای آن‌ها) ارائه می‌شود. مدل زبان بزرگ تصمیم می‌گیرد که کدام ابزار را با چه آرگومان‌هایی بر اساس درخواست کاربر فراخوانی کند.  
- **روند**: درخواست کاربر -> LLM -> **انتخاب ابزار** -> **اجرای ابزار** -> LLM (با خروجی ابزار) -> پاسخ نهایی.  
- **موارد استفاده**: بازیابی داده‌های زمان واقعی (آب و هوا، قیمت سهام)، انجام محاسبات، اجرای کد.

### **الگوی برنامه‌ریزی** (درس ۷)  
این الگو به عامل‌ها امکان می‌دهد تا وظایف پیچیده و چندمرحله‌ای را حل کنند.  
- **مفهوم**: عامل هدف سطح بالا را به رشته‌ای از زیرکارهای کوچک‌تر تقسیم می‌کند.  
- **رویکردها**:  
  - **تفکیک کار**: تقسیم «برنامه‌ریزی سفر» به «رزرو پرواز»، «رزرو هتل»، «کرایه خودرو».  
  - **برنامه‌ریزی تکراری**: بازبینی برنامه بر اساس خروجی مراحل قبلی (مثلاً اگر پرواز پر باشد، تاریخ دیگری انتخاب شود).  
- **پیاده‌سازی**: معمولاً شامل یک عامل “برنامه‌ریز” است که برنامه‌ای ساختاریافته (مثلاً JSON) تولید می‌کند که سپس توسط عامل‌های دیگر اجرا می‌شود.

## 4. اصول طراحی

هنگام طراحی عامل‌ها، سه بُعد را در نظر بگیرید:  
- **فضا**: عامل‌ها باید مردم و دانش را به هم متصل کنند، دسترسی‌پذیر و در عین حال نامحسوس باشند.  
- **زمان**: عامل‌ها باید از *گذشته* بیاموزند، در *حال* انگیزه‌های مرتبط ارائه دهند و برای *آینده* سازگار شوند.  
- **هسته**: عدم قطعیت را بپذیرند اما از طریق شفافیت و کنترل کاربر اعتماد ایجاد کنند.

## 5. خلاصه درس‌های کلیدی

- **درس ۱**: عامل‌ها سیستم هستند، نه فقط مدل. آن‌ها درک، استدلال و عمل می‌کنند.  
- **درس ۲**: Microsoft Agent Framework پیچیدگی‌های فراخوانی ابزار و مدیریت وضعیت را انتزاع می‌کند.  
- **درس ۳**: طراحی با تمرکز بر شفافیت و کنترل کاربر.  
- **درس ۴**: ابزارها «دست‌های» عامل هستند. تعریف طرح‌واره برای درک نحوه استفاده توسط LLM حیاتی است.  
- **درس ۷**: برنامه‌ریزی «وظیفه اجرایی» عامل است که امکان حل جریان‌های کاری پیچیده را فراهم می‌کند.
