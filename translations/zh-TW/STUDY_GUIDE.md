# AI 新手代理人指南暨課程摘要

本指南提供「AI 新手代理人」課程的摘要，並說明建立 AI 代理人的核心概念、框架與設計模式。

## 1. 代理人簡介

**什麼是 AI 代理人？**  
AI 代理人是擴展大型語言模型（LLM）能力的系統，賦予其存取**工具**、**知識**和**記憶**的能力。與僅根據訓練資料生成文字的標準 LLM 聊天機器人不同，AI 代理人能夠：  
- **感知**其環境（透過感測器或輸入）。  
- **推理**如何解決問題。  
- **行動**改變環境（透過執行機構或工具執行）。

**代理人核心元件：**  
- **環境**：代理人運作的空間（例如預訂系統）。  
- **感測器**：蒐集資訊的機制（例如讀取 API）。  
- **執行機構**：執行動作的機制（例如發送電子郵件）。  
- **大腦（LLM）**：負責規劃與決策該採取哪些行動的推理引擎。

## 2. 代理人框架

本課程採用 **Microsoft 代理人框架（MAF）** 及 **Azure AI Foundry Agent Service V2** 建立代理人：

| 元件 | 重點 | 適合用途 |
|-----------|-------|----------|
| **Microsoft 代理人框架** | 統一的 Python / C# SDK，涵蓋代理人、工具與工作流程 | 建立結合工具、多代理人工作流程及生產環境設計的代理人。 |
| **Azure AI Foundry Agent Service** | 托管雲端運行環境 | 安全且可擴展的部署，內建狀態管理、可觀察性和信任機制。 |

## 3. 代理人設計模式

設計模式可協助結構化代理人的運作，可靠地解決問題。

### **工具使用模式**（第 4 課）  
此模式使代理人能與外部世界互動。  
- **概念**：代理人會取得「結構」（可用功能清單及參數），LLM 根據使用者請求決定*呼叫哪個工具*及*用什麼參數*。  
- **流程**：使用者請求 -> LLM -> **工具選擇** -> **工具執行** -> LLM（帶工具輸出）-> 最終回應。  
- **應用情境**：擷取即時資料（天氣、股價）、執行計算、執行程式碼。

### **規劃模式**（第 7 課）  
此模式使代理人能解決複雜、多步驟的任務。  
- **概念**：代理人將高階目標拆解成一系列小任務。  
- **方法**：  
  - **任務分解**：將「規劃旅遊」拆成「訂機票」、「訂飯店」、「租車」。  
  - **迭代規劃**：根據先前步驟的結果重新評估計畫（例如航班客滿則改選其他日期）。  
- **實作**：通常包含一個「規劃者」代理人產出結構化計劃（例如 JSON），由其他代理人執行。

## 4. 設計原則

設計代理人時，考量三個面向：  
- **空間**：代理人應連結人與知識，易於接近且不干擾。  
- **時間**：代理人應從*過去*學習，於*當下*提供相關提示，並為*未來*調整。  
- **核心**：接受不確定性，並透過透明度與用戶控制建立信任。

## 5. 重要課程摘要

- **第 1 課**：代理人是系統，不只是模型。他們能感知、推理與行動。  
- **第 2 課**：Microsoft 代理人框架抽象化工具呼叫與狀態管理的複雜性。  
- **第 3 課**：設計時須重視透明度與用戶控制。  
- **第 4 課**：工具是代理人的「雙手」。結構定義對於 LLM 理解如何使用工具至關重要。  
- **第 7 課**：規劃是代理人的「執行功能」，使其能處理複雜工作流程。
