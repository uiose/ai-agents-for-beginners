# 初心者のためのAIエージェント - 学習ガイド＆コース概要

このガイドは「初心者のためのAIエージェント」コースの概要を提供し、AIエージェント構築のための重要な概念、フレームワーク、設計パターンを説明します。

## 1. AIエージェント入門

**AIエージェントとは何か？**  
AIエージェントは、Large Language Models（LLMs）の能力を拡張し、**ツール**、**知識**、および**記憶**へのアクセスを可能にするシステムです。単にトレーニングデータに基づいてテキストを生成する通常のLLMチャットボットとは異なり、AIエージェントは以下のことができます：  
- 環境を**感知**する（センサーや入力を介して）。  
- 問題解決のために**推論**する。  
- 環境を変えるために**行動**する（アクチュエータやツール実行を介して）。

**エージェントの主要構成要素：**  
- **環境**：エージェントが動作する空間（例：予約システム）。  
- **センサー**：情報を収集する仕組み（例：APIの読み取り）。  
- **アクチュエータ**：アクションを実行する仕組み（例：メール送信）。  
- **脳（LLM）**：どのアクションを取るか計画し決定する推論エンジン。

このコースでは、**Microsoft Agent Framework (MAF)** と **Azure AI Foundry Agent Service V2** を使ってエージェントを構築します：

| コンポーネント | フォーカス | 適応例 |
|---------------|------------|--------|

## 3. エージェント設計パターン

設計パターンは、エージェントが問題を信頼性を持って解決するための構造を助けます。

### **ツール使用パターン**（レッスン4）  
このパターンはエージェントが外部と対話することを可能にします。  
- **コンセプト**：エージェントに「スキーマ」（利用可能な関数とそのパラメータの一覧）が提供されます。LLMはユーザーの要求に基づき、*どの*ツールを*どの*引数で呼び出すか決定します。  
- **フロー**：ユーザーの要求 -> LLM -> **ツール選択** -> **ツール実行** -> LLM（ツールの出力付き） -> 最終応答。  
- **使用例**：リアルタイムデータの取得（天気、株価）、計算の実行、コードの実行。

### **プランニングパターン**（レッスン7）  
このパターンはエージェントが複雑で複数段階のタスクを解決することを可能にします。  
- **コンセプト**：エージェントは高レベルの目標をより小さなサブタスクのシーケンスに分解します。  
- **アプローチ**：  
  - **タスク分解**：「旅行を計画する」を「航空券予約」「ホテル予約」「レンタカー手配」に分割。  
  - **反復的プランニング**：前のステップの結果に基づき計画を再評価（例：便が満席なら別の日程を選ぶ）。  
- **実装**：多くは「プランナー」エージェントが構造化された計画（例：JSON）を作成し、それを他のエージェントが実行する形。

## 4. 設計原則

エージェントを設計するときは3つの次元を考慮してください：  
- **空間**：エージェントは人と知識をつなぎ、アクセス可能でありながら目立ちすぎない存在であるべき。  
- **時間**：エージェントは*過去*から学び、*現在*に適切なナッジを提供し、*未来*へ適応するべき。  
- **コア**：不確実性を受け入れつつ、透明性とユーザー制御により信頼を築く。

## 5. 重要レッスンのまとめ

- **レッスン1**：エージェントは単なるモデルではなくシステムである。感知し、推論し、行動する。  
- **レッスン2**：Microsoft Agent Frameworkはツール呼び出しと状態管理の複雑さを抽象化する。  
- **レッスン3**：透明性とユーザー制御を考慮して設計する。  
- **レッスン4**：ツールはエージェントの「手」である。LLMが使い方を理解するためにスキーマ定義が重要。  
- **レッスン7**：プランニングはエージェントの「実行機能」であり、複雑なワークフロー処理を可能にする。
