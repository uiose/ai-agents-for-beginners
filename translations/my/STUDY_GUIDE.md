# စတင်သည့်သူများအတွက် AI ကိုယ်စားလှယ်များ - သင်ခန်းစာလေ့လာလမ်းညွှန်နှင့် သင်တန်းအကျဉ်းချုပ်

ဤလမ်းညွှန်သည် "စတင်သည့်သူများအတွက် AI ကိုယ်စားလှယ်များ" သင်တန်း၏ အကျဉ်းချုပ်ကို ပံ့ပိုးပေးပြီး AI ကိုယ်စားလှယ်များ တည်ဆောက်ရာတွင်အဓိကဖြစ်သော အယူအဆများ၊ ဖွဲ့စည်းတည်ဆောက်မှုများနှင့် ဒီဇိုင်းပုံစံများကို ရှင်းပြသည်။

## ၁။ AI ကိုယ်စားလှယ်များသို့ နိဒါန်း

**AI ကိုယ်စားလှယ်များ란 무엇입니까?**  
AI ကိုယ်စားလှယ်များသည် စကားပြောသင်ကြားမှု မော်ဒယ်ကြီးများ (LLMs) ၏ အသွင်ပြောင်းမှုများဖြစ်ပြီး အတတ်ပညာများဖြစ်သော **ကိရိယာများ**၊ **အသိပညာ** နှင့် **မှတ်ဉာဏ်** အသုံးပြုခွင့်ကို ပေးသည်။ ပုံမှန် LLM chatbot များက သင်ကြားမှု ဒေတာအရ ဗီစာတစ်ခုထုတ်ပေးသော်လည်း၊ AI ကိုယ်စားလှယ်သည် -  
- ၎င်း၏ပတ်ဝန်းကျင်ကို **ခံယူ** နိုင်သည် (ဆင်ဆာများ သို့မဟုတ် အထွက်များမှတဆင့်)။  
- ပြဿနာဖြေရှင်းနည်းကို **ထင်မြင်** နိုင်သည်။  
- ပတ်ဝန်းကျင်ကို **လှုပ်ရှားစေ** နိုင်သည် (အက်ချ်နစ်များ သို့မဟုတ် ကိရိယာများထိန်းချုပ်ခြင်းမှတဆင့်)။

**ကိုယ်စားလှယ်၏ အဓိကအစိတ်အပိုင်းများ**  
- **ပတ်ဝန်းကျင်**: ကိုယ်စားလှယ် စစ်ဆင်ရေးလုပ်ဆောင်သည့် နေရာ (ဥပမာ၊ ကြိုတင်စာရင်းစနစ်)  
- **ဆင်ဆာများ**: အချက်အလက်စုဆောင်းရေးကိရိယာများ (ဥပမာ၊ API ဖတ်ရူခြင်း)  
- **အက်ချ်နစ်များ**: လုပ်ဆောင်မှုများ ပြုလုပ်ရန်ကိရိယာများ (ဥပမာ၊ အီးမေးလ်ပို့ခြင်း)  
- **ဦးနှောက် (LLM)**: လုပ်ဆောင်ရန် အရေးကြီးချက်များ စီမံချက်ဆွဲသော ထင်မြင်ချက်စက်ရုပ်

## ၂။ ကိုယ်စားလှယ်ဖွဲ့စည်းမှု

သင်တန်းသည် **Microsoft Agent Framework (MAF)** ကို **Azure AI Foundry Agent Service V2** နှင့်အတူ ကိုယ်စားလှယ်များ တည်ဆောက်ရာတွင် အသုံးပြုသည်။

| အစိတ်အပိုင်း | အာရုံစိုက်ရာ | အကောင်းဆုံး အသုံးပြုနိုင်သည့်နေရာ |
|-----------|-------|----------|
| **Microsoft Agent Framework** | ကိုယ်စားလှယ်များ၊ ကိရိယာများနှင့် အလုပ်စဉ်များအတွက် တစ်ဦးတည်းဖြစ်သော Python/C# SDK | ကိရိယာပါရှိသော ကိုယ်စားလှယ်များ၊ မျိုးစုံကိုယ်စားလှယ် အလုပ်စဉ်များနှင့် ထုတ်လုပ်ရေး ပုံစံများ တည်ဆောက်ရာတွင် အသုံးပြုသည်။ |
| **Azure AI Foundry Agent Service** | မန်နေဂျ်လုပ်ထားသော မိုးကောင်းကင်ကြား အချိန်ပြေး | အခြေအနေစီမံခန့်ခွဲမှု၊ ကြည့်ရှုနိုင်မှုနှင့် ယုံကြည်မှုပါသောလုံခြုံပြီး ဂဏန်းတိုးသော တပ်ဆင်မှု။ |

## ၃။ ကိုယ်စားလှယ် ဒီဇိုင်းပုံစံများ

ဒီဇိုင်းပုံစံများသည် ကိုယ်စားလှယ်များ၏ လုပ်ငန်းဆောင်တာကို တရားဝင်ပြုလုပ်ရာတွင် အခြေခံဖွဲ့စည်းထားပုံကို အကူအညီပေးသည်။

### **ကိရိယာ အသုံးပြုမှု ပုံစံ** (သင်ခန်းစာ ၄)  
ဤပုံစံသည် ကိုယ်စားလှယ်များကို အပြင်ပတ်ဝန်းကျင်နှင့် ဆက်သွယ်ခွင့်ပြုသည်။  
- **အယူအဆ**: ကိုယ်စားလှယ်အား "schema" (ရရှိနိုင်သည့်လုပ်ဆောင်ချက်များစာရင်းနှင့် ၎င်းတို့ ပါရာမီတာများ) ပေးသည်။ LLM သည် အသုံးပြုသူ၏ တောင်းဆိုမှုအရ *မည်သည့်* ကိရိယာအား *မည်သည့်* ဟန်ချက်ဖြင့် ခေါ်ဖို့ ဆုံးဖြတ်သည်။  
- **စီးဆင်းမှု**: အသုံးပြုသူ တောင်းဆိုမှု -> LLM -> **ကိရိယာ ရွေးချယ်ခြင်း** -> **ကိရိယာ အလုပ်လုပ်ခြင်း** -> LLM (ကိရိယာ ထုတ်လွှင့်ချက်ဖြင့်) -> နောက်ဆုံးဖြေကြားချက်  
- **အသုံးပြုမှုများ**: တိကျသော အချိန် ရက်စွဲဒေတာရယူခြင်း (ရာသီဥတု၊ စတော့ရှယ်ယာတန်ဖိုးများ), တွက်ချက်မှုများ ဆောင်ရွက်ခြင်း, ကိုဒ်များ လုပ်ဆောင်ခြင်း

### **စီမံချက် ပုံစံ** (သင်ခန်းစာ ၇)  
ဤပုံစံသည် ကိုယ်စားလှယ်များကို တောင့်တင်းပြီး အဆင့်များစွာသော လုပ်ငန်းများ ဖြေရှင်းနိုင်စေသည်။  
- **အယူအဆ**: ကိုယ်စားလှယ်သည် ဦးစီးသောရည်မှန်းချက်ကို ကြီးကြပ်သော လုပ်ငန်းငယ်ငယ်များအဖြစ် ခွဲထုတ်သည်။  
- **နည်းလမ်းများ**:  
  - **အလုပ်ခွဲခြမ်းခြင်း**: "ခရီးစဥ် စီမံခြင်း" ကို "လေယာဉ်လက်မှတ်booking", "ဟိုတယ် réservation", "ကားငှားခြင်း" အဖြစ် ခွဲခြမ်းသည်။  
  - **ပြန်လည်စဥ်းစားခြင်း**: ရှေ့ဆက်မှုပုံစံကို အရင် အဆင့်များ ထွက်ရှိချက်အရ ပြန်လည်ကြည့်သုံးခြင်း (ဥပမာ၊ လေယာဉ်လက်မှတ် ပြည့်နေပါက အခြားရက်ရွေးချယ်ခြင်း)  
- **တည်ဆောက်ခြင်း**: အများအားဖြင့် "Planner" ကိုယ်စားလှယ်တစ်ယောက် မှ စနစ်တကျ ဖော်ဆောင်သည့် စီမံချက် (ဥပမာ JSON) များကို အခြားကိုယ်စားလှယ်များက အကောင်အထည်ဖော်သည်။

## ၄။ ဒီဇိုင်းနိယာမများ

ကိုယ်စားလှယ်များဒီဇိုင်းလုပ်ရာတွင် အချက်သုံးချက်ကို စဉ်းစားရန် -  
- **နေရာ**: ကိုယ်စားလှယ်များသည် လူများနှင့် အသိပညာတို့အား ချိတ်ဆက်ပေးပြီး လွယ်ကူသော်လည်း မဆိုးရွားသော ပုံစံဖြင့် ရှိရမည်။  
- **အချိန်**: ကိုယ်စားလှယ်များသည် *အတိတ်*မှ သင်ယူပြီး၊ *ယခု* မှာ သင့်တော်သော သတိပေးချက်များပေးကာ၊ *အနာဂတ်* အတွက် ကိုက်ညီ ပြောင်းလဲနိုင်ရမည်။  
- **အခြေခံ**: မသေချာမှုကို လက်ခံပိုင်နိုင်ပြီး၊ ထွက်ဟန်ပြမှုနှင့် အသုံးပြုသူ ထိန်းချုပ်မှုအားဖြင့် ယုံကြည်မှု တည်ဆောက်ရမည်။

## ၅။ အဓိက သင်ခန်းစာများ အကျဉ်းချုပ်

- **သင်ခန်းစာ ၁**: ကိုယ်စားလှယ်များသည် မော်ဒယ်များသာမက စနစ်များ ဖြစ်ကြသည်။ ၎င်းတို့သည် ခံယူ၊ ထင်ထုပြီး လုပ်ဆောင်သည်။  
- **သင်ခန်းစာ ၂**: Microsoft Agent Framework သည် ကိရိယာခေါ်ဆိုမှုနှင့် အခြေအနေ စီမံခန့်ခွဲမှု၏ရှုပ်ထွေးမှုကို ဖုံးကွယ်ပေးသည်။  
- **သင်ခန်းစာ ၃**: ဖွဲ့စည်းရာတွင် ထွက်ဟန်ပြမှုနှင့် အသုံးပြုသူ ထိန်းချုပ်မှုတို့ကို စဉ်းစားရမည်။  
- **သင်ခန်းစာ ၄**: ကိရိယာများသည် ကိုယ်စားလှယ်၏ "လက်" ဖြစ်သည်။ Schema ဖော်ပြချက်သည် LLM သို့ မည်သို့ အသုံးချရမည်ကို နားလည်စေသည်။  
- **သင်ခန်းစာ ၇**: စီမံချက်သည် ကိုယ်စားလှယ်၏ "အမှုဆောင်လုပ်ငန်း" ဖြစ်ကာ ကြီးမားသော အလုပ်စဉ်များကို ကျက်စားနိုင်စေသည်။
