# AI 代理入门 - 学习指南与课程总结

本指南总结了“AI 代理入门”课程，并解释了构建 AI 代理的关键概念、框架和设计模式。

## 1. AI 代理介绍

**什么是 AI 代理？**  
AI 代理是通过赋予大型语言模型（LLM）访问**工具**、**知识**和**记忆**的能力，来扩展其功能的系统。与仅基于训练数据生成文本的普通 LLM 聊天机器人不同，AI 代理能够：  
- **感知**其环境（通过传感器或输入）。  
- **推理**如何解决问题。  
- **行动**以改变环境（通过执行器或工具执行）。

**代理的关键组成部分：**  
- **环境**：代理运行的空间（例如预订系统）。  
- **传感器**：收集信息的机制（例如读取 API）。  
- **执行器**：执行动作的机制（例如发送电子邮件）。  
- **大脑（LLM）**：计划和决定采取何种行动的推理引擎。

## 2. 代理框架

本课程使用 **Microsoft Agent Framework (MAF)** 和 **Azure AI Foundry Agent Service V2** 来构建代理：

| 组件 | 重点 | 适用场景 |
|-----------|-------|----------|
| **Microsoft Agent Framework** | 统一的 Python/C# SDK，适用于代理、工具和工作流 | 构建带工具支持的代理、多代理工作流及生产模式。 |
| **Azure AI Foundry Agent Service** | 托管云运行时 | 安全、可扩展的部署，内置状态管理、可观测性和信任机制。 |

## 3. 代理设计模式

设计模式有助于构建稳健的代理操作流程以解决问题。

### **工具使用模式**（课程第4课）  
该模式使代理能够与外部世界交互。  
- **概念**：向代理提供“schema”（可用函数及其参数列表）。LLM 根据用户请求决定调用*哪个*工具和传入*什么*参数。  
- **流程**：用户请求 -> LLM -> **工具选择** -> **工具执行** -> 带工具输出的 LLM -> 最终响应。  
- **应用场景**：获取实时数据（天气、股票价格）、执行计算、运行代码。

### **规划模式**（课程第7课）  
该模式使代理能够解决复杂的多步骤任务。  
- **概念**：代理将高级目标拆分成一系列较小的子任务。  
- **方法**：  
  - **任务分解**：将“计划旅行”拆分为“预订航班”、“预订酒店”、“租车”。  
  - **迭代规划**：基于前一步结果重新评估计划（例如，如果航班已满，选择其他日期）。  
- **实现**：通常包含一个“规划者”代理生成结构化计划（例如 JSON），由其他代理执行。

## 4. 设计原则

设计代理时考虑三个维度：  
- **空间**：代理应连接人与知识，既可访问又不干扰。  
- **时间**：代理应从*过去*学习，提供*当前*相关提示，并适应*未来*。  
- **核心**：面对不确定性，同时通过透明度和用户控制建立信任。

## 5. 关键课程总结

- **第1课**：代理是系统，不仅仅是模型。它们能够感知、推理和行动。  
- **第2课**：Microsoft Agent Framework 抽象了工具调用和状态管理的复杂性。  
- **第3课**：设计时应考虑透明度和用户控制。  
- **第4课**：工具是代理的“手”。定义 schema 对于 LLM 理解工具使用至关重要。  
- **第7课**：规划是代理的“执行功能”，使其能够应对复杂工作流。
