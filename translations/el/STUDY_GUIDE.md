# Πράκτορες Τεχνητής Νοημοσύνης για Αρχάριους - Οδηγός Μελέτης & Περίληψη Μαθήματος

Αυτός ο οδηγός παρέχει μια περίληψη του μαθήματος "Πράκτορες Τεχνητής Νοημοσύνης για Αρχάριους" και εξηγεί βασικές έννοιες, πλαίσια και σχεδιαστικά πρότυπα για την κατασκευή Πρακτόρων Τεχνητής Νοημοσύνης.

## 1. Εισαγωγή στους Πράκτορες Τεχνητής Νοημοσύνης

**Τι είναι οι Πράκτορες Τεχνητής Νοημοσύνης;**  
Οι Πράκτορες Τεχνητής Νοημοσύνης είναι συστήματα που επεκτείνουν τις δυνατότητες των Μεγάλων Γλωσσικών Μοντέλων (LLMs) δίνοντάς τους πρόσβαση σε **εργαλεία**, **γνώση** και **μνήμη**. Σε αντίθεση με ένα τυπικό chatbot LLM που απλώς παράγει κείμενο βασισμένο σε δεδομένα εκπαίδευσης, ένας Πράκτορας ΤΝ μπορεί να:  
- **Αντιλαμβάνεται** το περιβάλλον του (μέσω αισθητήρων ή εισόδων).  
- **Λογικεύει** για το πώς να λύσει ένα πρόβλημα.  
- **Δρα** ώστε να αλλάξει το περιβάλλον (μέσω εκτελεστών ενεργειών ή εκτέλεσης εργαλείων).

**Κύρια Συστατικά ενός Πράκτορα:**  
- **Περιβάλλον**: Ο χώρος όπου λειτουργεί ο πράκτορας (π.χ. σύστημα κρατήσεων).  
- **Αισθητήρες**: Μηχανισμοί για τη συλλογή πληροφοριών (π.χ. ανάγνωση ενός API).  
- **Εκτελεστές**: Μηχανισμοί για την ενέργεια (π.χ. αποστολή email).  
- **Εγκέφαλος (LLM)**: Η μηχανή λογικής που σχεδιάζει και αποφασίζει ποιες ενέργειες θα πραγματοποιηθούν.

## 2. Πλαίσια Πρακτόρων

Το μάθημα χρησιμοποιεί το **Microsoft Agent Framework (MAF)** με το **Azure AI Foundry Agent Service V2** για την κατασκευή πρακτόρων:

| Συστατικό | Εστίαση | Ιδανικό για |
|-----------|---------|-------------|
| **Microsoft Agent Framework** | Ενιαίο Python/C# SDK για πράκτορες, εργαλεία και ροές εργασίας | Κατασκευή πρακτόρων με εργαλεία, ροές εργασίας πολλαπλών πρακτόρων και πρότυπα παραγωγής. |
| **Azure AI Foundry Agent Service** | Διαχειριζόμενο περιβάλλον εκτέλεσης στο σύννεφο | Ασφαλής, κλιμακούμενη ανάπτυξη με ενσωματωμένη διαχείριση κατάστασης, παρατηρησιμότητα και αξιοπιστία. |

## 3. Σχεδιαστικά Πρότυπα για Πράκτορες

Τα σχεδιαστικά πρότυπα βοηθούν στη δομή του τρόπου λειτουργίας των πρακτόρων για αξιόπιστη επίλυση προβλημάτων.

### **Πρότυπο Χρήσης Εργαλείων** (Μάθημα 4)  
Αυτό το πρότυπο επιτρέπει στους πράκτορες να αλληλεπιδρούν με τον έξω κόσμο.  
- **Έννοια**: Ο πράκτορας παρέχεται με ένα "σχήμα" (λίστα διαθέσιμων λειτουργιών και των παραμέτρων τους). Το LLM αποφασίζει *ποιο* εργαλείο θα καλέσει και με *ποιες* παραμέτρους βάσει του αιτήματος του χρήστη.  
- **Ροή**: Αίτημα Χρήστη -> LLM -> **Επιλογή Εργαλείου** -> **Εκτέλεση Εργαλείου** -> LLM (με αποτέλεσμα εργαλείου) -> Τελική Απόκριση.  
- **Εφαρμογές**: Ανάκτηση δεδομένων σε πραγματικό χρόνο (καιρός, τιμές μετοχών), εκτέλεση υπολογισμών, εκτέλεση κώδικα.

### **Πρότυπο Σχεδιασμού** (Μάθημα 7)  
Αυτό το πρότυπο επιτρέπει στους πράκτορες να επιλύουν σύνθετα, πολυβηματικά καθήκοντα.  
- **Έννοια**: Ο πράκτορας σπάει έναν υψηλού επιπέδου στόχο σε μια αλληλουχία μικρότερων υποκαθηκόντων.  
- **Προσεγγίσεις**:  
  - **Αποσύνθεση Καθήκοντος**: Διαχωρισμός του "Σχεδίασε ένα ταξίδι" σε "Κράτηση πτήσης", "Κράτηση ξενοδοχείου", "Ενοικίαση αυτοκινήτου".  
  - **Επαναληπτικός Σχεδιασμός**: Επανεκτίμηση του σχεδίου βάσει των αποτελεσμάτων των προηγούμενων βημάτων (π.χ. αν η πτήση είναι πλήρης, επιλογή διαφορετικής ημερομηνίας).  
- **Υλοποίηση**: Συχνά περιλαμβάνει έναν "σχεδιαστή" πράκτορα που δημιουργεί ένα δομημένο σχέδιο (π.χ. σε JSON) το οποίο εκτελείται από άλλους πράκτορες.

## 4. Αρχές Σχεδιασμού

Κατά τον σχεδιασμό πρακτόρων, λάβετε υπόψη τρεις διαστάσεις:  
- **Χώρος**: Οι πράκτορες πρέπει να συνδέουν ανθρώπους και γνώση, να είναι προσβάσιμοι αλλά διακριτικοί.  
- **Χρόνος**: Οι πράκτορες πρέπει να μαθαίνουν από το *Παρελθόν*, να παρέχουν σχετικές ειδοποιήσεις στο *Τώρα* και να προσαρμόζονται για το *Μέλλον*.  
- **Κορμός**: Υιοθετήστε την αβεβαιότητα αλλά εδραιώστε την αξιοπιστία μέσω διαφάνειας και ελέγχου από τον χρήστη.

## 5. Περίληψη Βασικών Μαθημάτων

- **Μάθημα 1**: Οι πράκτορες είναι συστήματα, όχι μόνο μοντέλα. Αντιλαμβάνονται, λογικεύουν και δρουν.  
- **Μάθημα 2**: Το Microsoft Agent Framework απλοποιεί την πολυπλοκότητα της κλήσης εργαλείων και της διαχείρισης κατάστασης.  
- **Μάθημα 3**: Σχεδιάστε με διαφάνεια και έλεγχο από τον χρήστη.  
- **Μάθημα 4**: Τα εργαλεία είναι τα "χέρια" του πράκτορα. Ο ορισμός του σχήματος είναι κρίσιμος για να καταλάβει το LLM πώς να τα χρησιμοποιήσει.  
- **Μάθημα 7**: Ο σχεδιασμός είναι η "εκτελεστική λειτουργία" του πράκτορα, που του επιτρέπει να αντιμετωπίζει σύνθετες ροές εργασίας.
