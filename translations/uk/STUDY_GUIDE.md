# AI-агенти для початківців - Посібник і конспект курсу

This guide provides a summary of the "AI Agents for Beginners" course and explains key concepts, frameworks, and design patterns for building AI Agents.

## 1. Вступ до AI-агентів

**Що таке AI-агенти?**
AI-агенти — це системи, які розширюють можливості Large Language Models (LLMs), надаючи їм доступ до **інструментів**, **знань** та **пам'яті**. На відміну від стандартного чат-бота на основі LLM, який лише генерує текст на основі навчальних даних, AI-агент може:
- **Спостерігати** своє середовище (через сенсори або вхідні дані).
- **Міркувати** про те, як вирішити проблему.
- **Діяти**, щоб змінити середовище (через актуатори або виконання інструментів).

**Ключові компоненти агента:**
- **Середовище**: Простір, в якому працює агент (наприклад, система бронювання).
- **Сенсори**: Механізми збору інформації (наприклад, запити до API).
- **Актуатори**: Механізми виконання дій (наприклад, відправлення електронного листа).
- **Мозок (LLM)**: Двигун міркування, який планує та вирішує, які дії виконувати.

## 2. Фреймворки для агентів

The course uses **Microsoft Agent Framework (MAF)** with **Azure AI Foundry Agent Service V2** for building agents:

| Component | Focus | Best For |
|-----------|-------|----------|
| **Microsoft Agent Framework** | Уніфікований SDK на Python/C# для агентів, інструментів та робочих процесів | Побудова агентів з інструментами, багатоагентних робочих процесів і продакшн-шаблонів. |
| **Azure AI Foundry Agent Service** | Кероване хмарне середовище виконання | Безпечне, масштабоване розгортання з вбудованим управлінням станом, спостережуваністю та механізмами довіри. |

## 3. Шаблони проєктування для агентів

Шаблони проєктування допомагають структурувати роботу агентів для надійного вирішення задач.

### **Шаблон використання інструментів** (Урок 4)
Цей шаблон дозволяє агентам взаємодіяти з навколишнім світом.
- **Концепція**: Агенту надається "schema" (список доступних функцій і їхніх параметрів). LLM вирішує, *який* інструмент викликати і з *якими* аргументами на основі запиту користувача.
- **Потік**: Запит користувача -> LLM -> **Вибір інструмента** -> **Виконання інструмента** -> LLM (з результатом інструмента) -> Кінцева відповідь.
- **Сценарії використання**: Отримання даних у реальному часі (погода, ціни на акції), виконання обчислень, виконання коду.

### **Шаблон планування** (Урок 7)
Цей шаблон дозволяє агентам вирішувати складні багатокрокові завдання.
- **Концепція**: Агент розбиває мету високого рівня на послідовність менших підзадач.
- **Підходи**:
  - **Декомпозиція завдань**: Розбивання "Запланувати поїздку" на "Забронювати рейс", "Забронювати готель", "Орендувати автомобіль".
  - **Ітеративне планування**: Повторна оцінка плану на основі результатів попередніх кроків (наприклад, якщо рейс заповнений, обрати іншу дату).
- **Реалізація**: Часто це включає агента "Planner", який генерує структурований план (наприклад, JSON), який потім виконують інші агенти.

## 4. Принципи проєктування

При проєктуванні агентів враховуйте три виміри:
- **Простір**: Агенти повинні поєднувати людей і знання, бути доступними, але ненав'язливими.
- **Час**: Агенти повинні вчитися з *Минулого*, надавати доречні підштовхування у *Теперішньому* та адаптуватися для *Майбутнього*.
- **Ядро**: Приймайте невизначеність, але встановлюйте довіру через прозорість і контроль користувача.

## 5. Підсумок ключових уроків

- **Урок 1**: Агенти — це системи, а не просто моделі. Вони сприймають, міркують і діють.
- **Урок 2**: Microsoft Agent Framework абстрагує складнощі виклику інструментів та управління станом.
- **Урок 3**: Проєктуйте з урахуванням прозорості та контролю користувача.
- **Урок 4**: Інструменти — це "руки" агента. Визначення схеми має вирішальне значення для того, щоб LLM розумів, як їх використовувати.
- **Урок 7**: Планування — це "виконавча функція" агента, яка дозволяє йому вирішувати складні робочі процеси.
