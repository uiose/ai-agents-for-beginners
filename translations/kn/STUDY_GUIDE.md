# ಪ್ರಾರಂಭಿಕರಿಗೆ AI ಏಜೆಂಟ್‌ಗಳು - ಅಧ್ಯಯನ ಮಾರ್ಗದರ್ಶಿ ಮತ್ತು کورس ಸಾರಾಂಶ

ಈ ಮಾರ್ಗದರ್ಶಿ "AI ಏಜೆಂಟ್‌ಗಳು ಪ್ರಾರಂಭಿಕರಿಗೆ" کورಸಿನ ಸಾರಾಂಶ ಒದಗಿಸುತ್ತದೆ ಮತ್ತು AI ಏಜೆಂಟ್‌ಗಳನ್ನು ನಿರ್ಮಿಸಲು ಪ್ರಮುಖ ಪರಿಕಲ್ಪನೆಗಳು, ಫ್ರೇಮ್ವರ್ಕ್‌ಗಳು ಮತ್ತು ವಿನ್ಯಾಸ ಮಾದರಿಗಳನ್ನು ವಿವರಿಸುತ್ತದೆ.

## 1. AI ಏಜೆಂಟ್‌ಗಳಿಗೆ ಪರಿಚಯ

**AI ಏಜೆಂಟ್‌ಗಳು ಏನು?**
AI ಏಜೆಂಟ್‌ಗಳು ದೊಡ್ಡ ಭಾಷಾ ಮಾದರಿಗಳ (LLMs) ಸಾಮರ್ಥ್ಯವನ್ನು ವಿಸ್ತರಿಸುವ ವ್ಯವಸ್ಥೆಗಳಾಗಿವೆ, ಅವುಗಳಿಗೆ **ಕರಣಗಳು**, **ಜ್ಞಾನ**, ಮತ್ತು **ಸ್ಮೃತಿ**ಗೆ ಪ್ರವೇಶ ನೀಡುತ್ತವೆ. ತರಬೇತಿ ಡೇಟಾ ಆಧಾರಿತವಾಗಿ ಪಠ್ಯವನ್ನು ಮಾತ್ರ ಸೃಷ್ಟಿಸುವ ನಿಗೂಢ LLM ಚಾಟ್‌ಬಾಟ್‌ನಿಂದ ಭಿನ್ನವಾಗಿ, AI ಏಜೆಂಟ್‌ಗಳು:
- **ಪರಿಸರವನ್ನು ಗ್ರಹಿಸುವುದು** (ಸENSORಗಳು ಅಥವಾ ಇನ್ಪುಟ್‌ಗಳ ಮೂಲಕ).
- **ಸಮಸ್ಯೆ ಪರಿಹಾರಕ್ಕೆ ಯುಕ್ತಿಉಪಯೋಗಿಸುವುದು**.
- **ಪರಿಸರವನ್ನು ಬದಲಾಯಿಸಲು ಕ್ರಮವಹಿಸುವುದು** (ಆಕ್ಟುಯೇಟರ್‌ಗಳು ಅಥವಾ ಕ್ರಮ ನಿರ್ವಹಣೆ ಮೂಲಕ).

**ಏಜೆಂಟ್‌ನ ಪ್ರಮುಖ ಘಟಕಗಳು:**
- **ಪರಿಸರ**: ಏಜೆಂಟ್ ಕಾರ್ಯ ನಿರ್ವಹಿಸುವ ಸ್ಥಳ (ಉದಾ: ಬುಕಿಂಗ್ ವ್ಯವಸ್ಥೆ).
- **ಸೆನ್ಸಾರ್‌ಗಳು**: ಮಾಹಿತಿ ಸಂಗ್ರಹಿಸುವ ಯಂತ್ರಗಳು (ಉದಾ: API ಓದುವುದು).
- **ಆಕ್ಟುಯೇಟರ್‌ಗಳು**: ಕ್ರಿಯೆಗಳನ್ನು ನೆರವೇರಿಸುವ ಯಂತ್ರಗಳು (ಉದಾ: ಇಮೇಲ್ ಕಳುಹಿಸುವುದು).
- **ಮಸ್ತಿಷ್ಕ (LLM)**: ಕ್ರಮ ಯೋಜನೆ ಮಾಡುವ ಮತ್ತು ಕ್ರಮಗಳನ್ನು ನಿರ್ಧರಿಸುವ ಯಂತ್ರ.

## 2. ಏಜೆಂಟ್ ಫ್ರೇಮ್ವರ್ಕ್‌ಗಳು

ಈ کورಸಿನಲ್ಲಿ ಏಜೆಂಟ್ ನಿರ್ಮಿಸಲು **Microsoft Agent Framework (MAF)** ಮತ್ತು **Azure AI Foundry Agent Service V2** ಬಳಸಲಾಗಿದೆ:

| ಘಟಕ | ಗಮನ केंद्र | ಅತ್ಯುತ್ತಮ ಬಳಕೆ |
|-------|-------------|-----------------|
| **Microsoft Agent Framework** | ಏಜೆಂಟ್‌ಗಳು, ಕರಣೆಗಳು, ಮತ್ತು ವರ್ಕ್‌ಫ್ಲೋಗಳಿಗೆ ಏಕೀಕೃತ Python/C# SDK | ಸಲಕರಣೆಗಳೊಂದಿಗೆ ಏಜೆಂಟ್‌ಗಳನ್ನು ನಿರ್ಮಿಸುವುದು, ಬಹು ಏಜೆಂಟ್ ವರ್ಕ್‌ಫ್ಲೋಗಳು, ಮತ್ತು ಉತ್ಪಾದನಾ ಮಾದರಿಗಳು. |
| **Azure AI Foundry Agent Service** | ನಿರ್ವಹಿತ ಕ್ಲೌಡ್ ರನ್‌ಟೈಮ್ | ಸುರಕ್ಷಿತ, ಕೇಳಿ ಹಚ್ಚುವ ಹಂತ ನಿರ್ವಹಣೆ, ಪಾರ್ದರ್ಶಕತೆ ಮತ್ತು ನಂಬಿಕೆ ಹೊಂದಿರುವ ತಂತ್ರಾಂಶ ವಿತರಣೆ. |

## 3. ಏಜೆಂಟಿ ವಿನ್ಯಾಸ ಮಾದರಿಗಳು

ವಿನ್ಯಾಸ ಮಾದರಿಗಳು ಏಜೆಂಟ್‌ಗಳು ವಿಶ್ವಾಸಾರ್ಹವಾಗಿ ಸಮಸ್ಯೆಗಳನ್ನು ಪರಿಹರಿಸುವುದಕ್ಕೆ ಹೇಗೆ ಕಾರ್ಯನಿರ್ವಹಿಸಬೇಕೆಂದು ರಚನೆ ಮಾಡಲು ಸಹಾಯ ಮಾಡುತ್ತವೆ.

### **ಕರಣ ಬಳಕೆ ಮಾದರಿ** (ಪಾಠ 4)
ಈ ಮಾದರಿ ಏಜೆಂಟ್‌ಗಳು ಹೊರಗಿನ ಜಗತ್ತಿನೊಂದಿಗೆ ಸಂವಹನ ಮಾಡಲು ಅನುಮತಿಸುತ್ತದೆ.
- **ಪರಿಕಲ್ಪನೆ**: ಏಜೆಂಟ್‌ಗೆ ಲಭ್ಯವಿರುವ ಕಾರ್ಯಗಳ ಮತ್ತು ಅವುಗಳ ಪರಿಮಾಣಗಳ ಎಂಬ "ವಿವರಣೆ" ನೀಡಲ್ಪಡುತ್ತದೆ. ಬಳಕೆದಾರರ ವಿನಂತಿಯನ್ನು ಆಧರಿಸಿ LLM ಯಾವ கரಣವನ್ನು ಕರೆಸುವುದು ಮತ್ತು ಯಾವಲ್ಲಾ ವಾದಗಳನ್ನು ಬಳಸುವುದು ಎಂದು ನಿರ್ಧರಿಸುತ್ತದೆ.
- **ಪ್ರವಾಹ**: ಬಳಕೆದಾರ ವಿನಂತಿ -> LLM -> **ಕರಣ ಆಯ್ಕೆ** -> **ಕರಣ ಕಾರ್ಯಾಚರಣೆ** -> LLM (ಕರಣ ಔಟ್ಪುಟ್ೊಂದಿಗೆ) -> ಅಂತಿಮ ಪ್ರತಿಕ್ರಿಯೆ.
- **ಬಳಕೆದಾರ ಪ್ರಕರಣಗಳು**: ನೈಜ-ಸಮಯದ ಡೇಟಾ ಪಡೆಯುವುದು (ವಾತಾವರಣ, ಷೇರು ಬೆಲೆಗಳು), ಗಣನೆ ಮಾಡುವುದು, ಕೋಡ್ ಕಾರ್ಯಾಚರಣೆ.

### **ಯೋಜನೆ ಮಾದರಿ** (ಪಾಠ 7)
ಈ ಮಾದರಿ ಏಜೆಂಟ್‌ಗಳು ಸಂಕೀರ್ಣ, ಬಹು ಹಂತದ ಕಾರ್ಯಗಳನ್ನು ಪರಿಹರಿಸಲು ಸಹಾಯ ಮಾಡುತ್ತದೆ.
- **ಪರಿಕಲ್ಪನೆ**: ಏಜೆಂಟ್ ಮೇಲ್ನೋಟದ ಗುರಿಯನ್ನು ಸಣ್ಣ ಉಪಕಾರ್ಯಗಳ ಸರಣಿಯಾಗಿ ವಿಭಜಿಸುತ್ತದೆ.
- **ವಿಧಾನಗಳು**:
  - **ಕಾರ್ಯ ವಿಭಜನೆ**: "ಪ್ರಯಾಣ ಯೋಜನೆ ಮಾಡಿ" ಅನ್ನು "ವಿಮಾನ ಬುಕ್ ಮಾಡಿ", "ಹೋಟೆಲ್ ಬುಕ್ ಮಾಡಿ", "ಕಾರ್ ಬಾಡಿಗೆಗೆ ತೆಗೆ" ಎಂದು ವಿಭಜಿಸುವುದು.
  - **ಪುನರ್ಮೂಲ್ಯಮಾಪನ ಯೋಜನೆ**: ಹಿಂದಿನ ಹಂತಗಳ ಔಟ್ಪುಟ್ ಆಧರಿಸಿ ಯೋಜನೆಯನ್ನು ಮರುಪರಿಶೀಲಿಸುವುದು (ಉದಾ: ವಿಮಾನ ತುಂಬಿದರೆ, ಬೇರೆ ದಿನಾಂಕ ಆರಿಸಿ).
- **ಅಮಲಾದೇಶ**: ಸಾಮಾನ್ಯವಾಗಿ "ಯೋಜನಾಕ" ಏಜೆಂಟ್ ಒಂದು ರಚನಾತ್ಮಕ ಯೋಜನೆಯನ್ನು (ಉದಾ: JSON) ತಯಾರಿಸಿ, ಅದನ್ನು ಇತರ ಏಜೆಂಟ್‌ಗಳು ನಿರ್ವಹಿಸುತ್ತವೆ.

## 4. ವಿನ್ಯಾಸ ಸಿದ್ಧಾಂತಗಳು

ಏಜೆಂಟ್‌ಗಳನ್ನು ವಿನ್ಯಾಸ ಮಾಡಲು ಮೂರು ಆಯಾಮಗಳನ್ನು ಪರಿಗಣಿಸಿ:
- **ಸ್ಪೇಸ್**: ಏಜೆಂಟ್‌ಗಳು ಜನರನ್ನು ಮತ್ತು ಜ್ಞಾನವನ್ನು ಸಂಪರ್ಕಿಸಬೇಕು, ಪ್ರಾಪ್ತಿಯಾಗುವಷ್ಟೆ ಆದರೆ ಅತಿ ಚುಟುಕು ಇರಬೇಕು.
- **ಸಮಯ**: ಏಜೆಂಟ್‌ಗಳು *ಹಿಂದಿನ* ಅನುಭವದಿಂದ ಕಲಿಯಬೇಕು, *ಪ್ರಸ್ತುತ* ಮಾಹಿತಿಯನ್ನು ಸರಿಯಾದ ಸಮಯದಲ್ಲಿ ನೀಡಬೇಕು, ಮತ್ತು *ಭವಿಷ್ಯ*ಗಾಗಿ ಹೊಂದಾಣಿಕೆ ಹೊಂದಬೇಕು.
- **ಮೂಲ**: ಅನುಮಾನವನ್ನು ಸ್ವೀಕರಿಸಿ ಆದರೆ ಪಾರದರ್ಶಕತೆಯಿಂದ ಮತ್ತು ಬಳಕೆದಾರನ ನಿಯಂತ್ರಣದಿಂದ ನಂಬಿಕೆ ನಿರ್ಮಿಸಬೇಕು.

## 5. ಪ್ರಮುಖ ಪಾಠಗಳ ಸಾರಾಂಶ

- **ಪಾಠ 1**: ಏಜೆಂಟ್‌ಗಳು ವ್ಯವಸ್ಥೆಗಳು, ಮಾತ್ರ ಮಾದರಿಗಳು ಅಲ್ಲ. ಅವುಗಳು ಗ್ರಹಿಸಬೇಕು, ಯುಕ್ತಿಯಾಗಿರಬೇಕು, ಮತ್ತು ಕ್ರಮವಹಿಸಬೇಕು.
- **ಪಾಠ 2**: Microsoft Agent Framework ಕರಣೆ ಕರೆಸುವುದು ಮತ್ತು ಸ್ಥಿತಿ ನಿರ್ವಹಣೆಯ ಸನ್ನಿವೇಶವನ್ನು ಸರಳಗೊಳಿಸುತ್ತದೆ.
- **ಪಾಠ 3**: ಪಾರದರ್ಶಕತೆ ಮತ್ತು ಬಳಕೆದಾರ ನಿಯಂತ್ರಣದೊಂದಿಗೆ ವಿನ್ಯಾಸ ಮಾಡಿ.
- **ಪಾಠ 4**: ಕರಣೆಗಳು ಏಜೆಂಟ್‌ನ "ಹಸ್ತಗಳು". ಕಾರಣ ಬಳಕೆಗಾಗಿ LLMಿಗೆ ಸ್ಕೀಮಾ ವಿವರಣೆ ಮುಖ್ಯ.
- **ಪಾಠ 7**: ಯೋಜನೆ ಏಜೆಂಟ್‌ನ "ನಿರ್ವಾಹಕ ಕಾರ್ಯ" ಆಗಿದ್ದು, ಸಂಕೀರ್ಣ ವರ್ಕ್‌ಫ್ಲೋಗಳಿಗೆ ಪರಿಹಾರ ನೀಡುತ್ತದೆ.
