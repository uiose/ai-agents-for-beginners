# Demo Results

这份文档整理了当前原型的几条代表性测试结果，可以直接作为组会中的“初步实验结果”。

## 测试方式

使用命令：

```powershell
.\venv\Scripts\python .\lab\course_qa_agent.py --query "<问题>"
```

知识库范围：

- `translations/zh-CN/02-explore-agentic-frameworks/README.md`
- `translations/zh-CN/03-agentic-design-patterns/README.md`
- `translations/zh-CN/04-tool-use/README.md`
- `translations/zh-CN/02-explore-agentic-frameworks/azure-ai-foundry-agent-creation.md`

## 案例 1：成功案例

问题：
什么是工具使用设计模式？

结果摘要：

- 系统能够命中 `04-tool-use/README.md` 中与工具使用设计模式直接相关的定义段落。
- 最终回答结构清晰，能概括定义和关键组成部分。
- 回答结尾给出了来源文件。

评价：

- 这是当前最稳定的一个问题。
- 说明“单 Agent + 本地检索工具 + 来源输出”的最小链路已经跑通。

## 案例 2：中等效果

问题：
Microsoft Agent Framework 和 Azure AI Agent Service 有什么区别？

结果摘要：

- 系统能够定位到框架差异相关内容，并给出总体区别。
- 回答能覆盖“框架”和“服务”的定位差异。
- 但回答仍然偏概括，没有把差异拆得足够细。

评价：

- 这个案例说明系统已经具备“概括性问答”的能力。
- 但对于对比类问题，当前检索和组织答案的能力还不够精细。

## 案例 3：边界案例

问题：
在 Azure AI Foundry 中创建 agent 时需要关注哪些关键元素？

结果摘要：

- 系统返回了客户端定义、代理定义、插件集成等要点。
- 但整体回答较短，也不够完整。
- 从检索结果看，当前命中的是代码片段和工具说明，相关证据还不够集中。

评价：

- 这个案例适合作为组会里的“当前问题”。
- 说明在结构化流程类问题上，当前轻量检索还不够稳定。

## 案例 4：解释性问题

问题：
为什么做课程资料问答时要用检索增强，而不是只靠大模型直接回答？

结果摘要：

- 系统能回答出准确性、知识更新和用户体验等理由。
- 结果整体合理，能够支撑“为什么做 RAG”的动机说明。
- 但回答中有些内容还是偏泛化，不完全是课程文档中的原句逻辑。

评价：

- 这个问题适合放在方法动机部分。
- 它能帮助解释为什么当前要做“检索增强问答”。

## 当前可汇报结论

- 已完成课程资料问答 Agent 的最小闭环验证。
- 系统已能针对部分定义类问题给出较稳定回答，并输出来源。
- 对比类、流程类问题仍存在回答泛化和检索不稳定的问题。
- 下一步应优先升级检索模块，并补充小规模评测样例。

## 组会上可以怎么说

我当前不是要证明系统已经做得很好，而是要证明最小原型已经跑通，并且我已经开始看到它在哪些问题上有效、又会在哪些问题上出错。  
从这个角度看，现在的结果是有价值的，因为它已经能支撑下一步的优化方向。
