# 你的 Agent 学习进度路线图

## 当前困境诊断

你说"不知道怎么开始写代码"，这是因为你现在处于：

```
被动学习（看代码）
     ↓
无法转化为能力（不会修改）
     ↓
更加无法创新（不会从零写）
```

这是一条死路。我们要换一条：
```
复制运行现成代码 ✓ 能马上看到效果
     ↓
修改参数、改数据结构 ✓ 理解每个部分的作用
     ↓
理解代码逻辑 ✓ 知道为什么这样写
     ↓
能扩展功能 ✓ 可以加新工具、新特性
     ↓ (经过3-4周的积累)
     ↓
能从零写新代码 ✓ 真正的理解
```

---

## 具体时间安排

### Week 1：从修改开始（不是从零开始）

| 日期 | 任务 | 时间 | 检查点 |
|------|------|------|--------|
| Day 1 | 安装、运行、理解代码结构 | 30分钟 | 代码能跑，见到 Agent 回答 |
| Day 2 | 练习1：改工具描述，对比效果 | 20分钟 | 理解 Schema 的作用 |
| Day 3 | 练习2：添加新课程 | 15分钟 | 理解数据结构 |
| Day 4 | 练习3：错误处理，看 Agent 怎么应对 | 25分钟 | 知道为什么需要异常捕捉 |
| Day 5 | 练习4：权限检查，理解访问控制 | 25分钟 | 理解第2层的作用 |
| Day 6 | 练习5：多轮对话，保存会话 | 30分钟 | 理解状态管理 |
| Day 7 | 总结 + 自由实验 | 1小时 | 自己尝试改2-3个地方 |

**本周目标**：不是"完全理解"，而是"知道每个部分做什么"。

---

### Week 2-3：从修改到创新

不急着从零写。继续改现在的代码，但加复杂度。

| 周次 | 任务 | 目标 |
|------|------|------|
| W2-1 | 添加新工具（比如"获取课程难度评分"） | 理解怎样定义新工具 |
| W2-2 | 添加日志详情（记录 Agent 的思考过程） | 理解第5层的重要性 |
| W2-3 | 添加 API 调用（连接真实数据而不是模拟数据） | 理解工具与外部系统的交互 |
| W3-1 | 做一个小项目（比如"代码审查小助手"） | 在新场景里应用已学知识 |

---

### Week 4+：开始写新代码

这时候，你会发现自己能做的事，比"从零开始写"更多。

## 答疑

**Q：我担心 API 费用**  
A：这个示例应该花费 $ < 0.01。不用担心。

**Q：我不确定是否理解正确**  
A：修改代码、看结果、对比预期。不需要完全理解，坚决执行比纠结理论更重要。

**Q：练习 3 里如果代码报错怎么办**  
A：截图发给我，或者直接问。这是调试过程的一部分。

**Q：这套练习需要多久**  
A：按部就班做，第一周 3-4 小时就够了。不用每天一个，可以一天做2-3个。

## 长期目标

3个月后，你应该能：
- [  ] 自己设计和实现一个简单的 Agent
- [  ] 理解代码中每个函数的职责
- [  ] 能为 Agent 添加新工具
- [  ] 能诊断 Agent 为什么给出错误答案
- [  ] 能在 Web、CLI、Serverless 等环境中运行 Agent

这些都不是凭空掌握的。都是通过**"修改现成代码、看效果、理解原因"**逐步积累的。
