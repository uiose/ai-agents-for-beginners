# 组会汇报 5 页 PPT 文案

## 第 1 页：研究目标

标题：
课程资料问答辅助 Agent 的最小原型设计与验证

可直接讲的内容：

我当前的工作重点，是先把 Agent 问题收缩成一个单任务场景，也就是面向课程资料和技术文档的问答辅助。  
输入是用户提出的问题，输出是结合相关资料生成的回答，并尽量给出来源。  
现阶段目标不是做复杂的通用多 Agent 系统，而是先验证一个最小可运行链路是否能够跑通。

这一页建议放 3 个点：

- 任务定义：课程资料问答
- 输入输出：问题到带来源回答
- 当前目标：验证最小可运行原型

## 第 2 页：方法设计

标题：
系统方案与技术路线

可直接讲的内容：

当前原型采用的是单 Agent 架构，核心思路是“大模型 + 本地检索增强 + 简单工具调用”。  
我用 Microsoft Agent Framework 的 Agent 接口搭建主流程，并实现了两个工具：一个是课程资料检索工具，用来从本地文档中提取相关上下文；另一个是资料列表工具，用来显式告诉 Agent 当前可访问哪些文档。  
在提示词层面，我约束 Agent 必须优先检索、尽量基于检索结果回答、并在回答结尾给出来源。

这一页建议放流程图文字版：

用户问题 -> 检索工具 -> 相关课程片段 -> Agent 组织答案 -> 输出回答与来源

这一页建议放 3 个点：

- 框架：Microsoft Agent Framework
- 检索：本地课程文档分块 + 关键词匹配
- 约束：只基于检索结果回答并输出来源

## 第 3 页：当前实现

标题：
已完成的最小原型

可直接讲的内容：

目前已经完成了一个可以运行的最小原型，代码放在 `lab/course_qa_agent.py`。

这一页建议放 4 个点：

- 已支持本地课程资料加载
- 已实现中文检索工具
- 已完成单 Agent 问答链路
- 已支持来源输出和交互式运行

## 第 4 页：初步实验结果

标题：
Demo 与初步观察

可直接讲的内容：

我选了几条代表性问题做了初步测试。  
例如在“什么是工具使用设计模式”这个问题上，系统能够较稳定地检索到 `04-tool-use/README.md` 中的相关段落，并生成结构化回答，还能给出来源。  
在“Microsoft Agent Framework 和 Azure AI Agent Service 有什么区别”这个问题上，系统也能给出总体性的差异说明，但回答还偏概括，不够细化。  
在“Azure AI Foundry 中创建 agent 时需要关注哪些关键元素”这个问题上，当前结果还比较粗糙，说明检索和答案组织还有提升空间。

- 失败或边界案例：Foundry 创建 agent 关键元素，结果较泛化

## 第 5 页：问题与下一步计划

标题：
当前问题与后续工作

可直接讲的内容：

当前主要问题有两个。  
第一，检索层还是轻量级关键词匹配，面对一些更抽象的问题时，相关段落排序不够稳定。  
第二，虽然已经能输出来源，但来源粒度还比较粗，离标准 RAG 系统还有距离。  
下一步我准备沿两个方向推进：一是把当前本地检索升级成更标准的检索方案，比如 Azure AI Search 或向量检索；二是补充一个小型测试集，对回答质量和来源一致性做更系统的评估。

这一页建议放 3 个点：

- 问题 1：检索精度有限
- 问题 2：回答粒度和可解释性还有待提升
- 下一步：升级为标准 RAG + 补评测样例
